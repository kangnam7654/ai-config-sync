사용자는 1인 개발자로 Python + Bash + iOS(SwiftUI) + Rust(Axum) + Next.js/TypeScript 풀스택 개발을 하고 있다. 플랜을 작성할 때 마일스톤 구조, 우선순위(P0/P1/P2), 크리티컬 패스, 리스크 분석까지 잘 갖추는 편이다. 초기에는 done-when을 빠뜨리는 경향이 있었으나, 피드백 후 명령어 기반 검증(pytest 개수, grep 패턴, exit code, curl 테스트)을 정확하게 작성한다. 배포/정리 단계도 초기에 누락하는 경향이 있었으나 피드백 후 잘 포함시킨다. 가정(assumptions)과 롤백 계획도 피드백 후 잘 반영한다. 병렬 가능 태스크 식별도 명시적으로 하게 됨. 2026-03 기준 PASS 수준(8.60)의 플랜을 초기 제출에서 바로 달성함. 남은 경미한 개선 포인트: (1) 리스크 테이블의 Rollback 컬럼을 N/A로 채우는 경향 -- 특히 Impact High 리스크에는 배포 롤백/feature flag 같은 구체적 방안 필요 (2) 마일스톤 합산 일수가 가용 영업일을 초과할 때 크로스-마일스톤 병렬 배치를 일정 계산에 명시적으로 반영하지 않는 경향.
