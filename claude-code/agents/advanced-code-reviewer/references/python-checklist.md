# Python Review Checklist

Detailed review rules for Python code. Loaded by code-reviewer when diff contains `.py` or `.pyi` files.

## File Classification

| Category | Detection Rule | Adjustments |
|---|---|---|
| **production** | Not in `tests/`, not generated, not a notebook | Full checklist |
| **test** | Path contains `tests/` or `test_` prefix or `_test.py` suffix | Skip P-FUNC-LENGTH (allow up to 100 lines), P-PARAM-COUNT (allow up to 8), P-NO-PRINT. Allow bare `assert`. |
| **generated** | Path contains `migrations/`, `*_pb2.py`, `*_pb2_grpc.py`, `*_generated.py`, or header contains `# Generated by` / `# DO NOT EDIT` | Skip entirely. Log: `[SKIP] {filepath}: generated code.` |
| **notebook** | `.ipynb` converted to `.py` | Review code cells only. Allow `print()` and inline plots. |

## Static Analysis Commands

```bash
uv run python -m ruff check --select=ALL --output-format=concise -- <files>
uv run python -m mypy --strict --no-error-summary -- <files>
uv run python -m bandit -q -- <files>
```

If all three unavailable: `[LIMITATION] No static analysis tools available. Manual-only review.`
If mypy only unavailable: `[LIMITATION] mypy unavailable. Type hints verified by manual inspection only.`

## Review Rules

### CRITICAL â€” Security

| ID | Rule | Bad | Good |
|---|---|---|---|
| S-SQL | No f-strings or `.format()` in SQL queries | `f"SELECT * FROM t WHERE id={uid}"` | `cursor.execute("SELECT * FROM t WHERE id=%s", (uid,))` |
| S-CMD | No unvalidated input in `subprocess` or `os.system` | `subprocess.run(f"ls {user_input}", shell=True)` | `subprocess.run(["ls", validated_path])` |
| S-PATH | No user-controlled paths without validation | `open(user_path)` | `resolved = Path(user_path).resolve(); assert resolved.is_relative_to(base_dir)` |
| S-EVAL | No `eval()`, `exec()`, `compile()` with external input | `eval(request.body)` | `json.loads()` or `ast.literal_eval()` |
| S-DESER | No `pickle.load()`, `yaml.load()` without SafeLoader, `marshal.load()` on untrusted data | `yaml.load(data)` | `yaml.safe_load(data)` |
| S-SECRET | No hardcoded secrets | `API_KEY = "sk-..."` | `API_KEY = os.environ["API_KEY"]` |
| S-CRYPTO | No MD5/SHA1 for security purposes | `hashlib.md5(password)` | `hashlib.sha256(password)` or `bcrypt` |

### CRITICAL â€” Exception Handling

| ID | Rule | Detail |
|---|---|---|
| E-BARE | No bare `except:` or `except Exception:` without re-raise or logging | Must (a) catch specific exception, (b) log via `logging.exception()`, or (c) re-raise |
| E-RERAISE | Maximum 1 re-raise per function | 2+ re-raises = split the function |
| E-SWALLOW | No empty `except` blocks | `except ValueError: pass` forbidden. At minimum log a warning. |
| E-CONTEXT | Use context managers for all closeable resources | `open()`, `Lock()`, `Connection()`, `Session()` must use `with` |

### HIGH â€” Type Hints

| ID | Rule | Detail |
|---|---|---|
| T-PUBLIC | All public functions must have complete type annotations | Every parameter + return type. "Public" = no `_` prefix. |
| T-PRIVATE | Private functions must have return type at minimum | Parameter types recommended but not required |
| T-ANY | `Any` forbidden in public function signatures | Use `object`, `TypeVar`, `Protocol`, or `Union`. `Any` allowed in private helpers and stubs only. |
| T-OPTIONAL | Use `X | None` (3.10+) or `Optional[X]` for nullable params | Never use `None` default without declaring type as optional |
| T-CAST | `cast()` requires a justifying comment | Every `cast(Type, value)` needs preceding or inline comment |
| T-IGNORE | `# type: ignore` must specify error code | `# type: ignore` alone forbidden. Use `# type: ignore[error-code]` |

**Python 2 compatibility**: If file contains `from __future__ import`, `six.`, `sys.version_info` checks â€” flag as `[TECH-DEBT]`. Skip T-PUBLIC, T-OPTIONAL, P-WALRUS, P-FSTRING for that file.

### HIGH â€” Pythonic Patterns

| ID | Rule | Bad | Good |
|---|---|---|---|
| P-LISTCOMP | Use comprehensions over manual loops for simple transforms | `result = []; for x in items: result.append(x.name)` | `result = [x.name for x in items]` |
| P-ISINSTANCE | Use `isinstance()` not `type() ==` | `type(x) == int` | `isinstance(x, int)` |
| P-ENUM | Use `Enum` not magic numbers/strings | `status = 1` | `status = Status.ACTIVE` |
| P-MUTABLE-DEFAULT | No mutable default arguments | `def f(x=[]):` | `def f(x: list[int] | None = None):` |
| P-FSTRING | Use f-strings over `%` formatting and `.format()` | `"hello %s" % name` | `f"hello {name}"` |
| P-WALRUS | Use walrus operator where it eliminates redundant calls (3.8+) | `m = re.match(p, s); if m:` | `if m := re.match(p, s):` |
| P-IS-NONE | Use `is None` / `is not None` for None checks | `x == None` | `x is None` |
| P-SHADOW | Do not shadow builtins | `list = [1, 2]` | `items = [1, 2]` |

### HIGH â€” Code Metrics (production files only)

| ID | Rule | Threshold |
|---|---|---|
| P-FUNC-LENGTH | Function body too long | > 50 lines (> 100 for test files) |
| P-PARAM-COUNT | Too many parameters | > 5 (> 8 for test files). Suggest dataclass or TypedDict. |
| P-NESTING | Nesting too deep | > 4 levels of indentation |
| P-NO-PRINT | No `print()` in production code | Use `logging`. Allowed in test files, CLI `__main__`, notebook cells. |

### HIGH â€” Async Code

Apply only to files containing `async def`, `await`, `asyncio.`, or `trio.`.

| ID | Rule | Detail |
|---|---|---|
| A-BLOCKING | No blocking calls inside async functions | `time.sleep()`, `open()` without `aiofiles`, `requests.get()` forbidden. Use `asyncio.sleep()`, `aiofiles.open()`, `httpx.AsyncClient`. |
| A-FIRE-FORGET | No fire-and-forget coroutines | `asyncio.create_task(coro())` must be stored in a variable. Bare `coro()` without `await` forbidden. |
| A-SYNC-IN-ASYNC | No `asyncio.run()` inside an already-running loop | Use `await` directly or `loop.create_task()` |
| A-CANCEL | Cancellation-safe resource cleanup | `try/finally` or `async with` around resources in cancellable async functions |
| A-GATHER | `asyncio.gather()` must use `return_exceptions=True` or wrap in try/except | Unhandled exceptions in gathered tasks crash silently |

### MEDIUM â€” Best Practices

| ID | Rule |
|---|---|
| B-IMPORT-ORDER | Imports in order: stdlib, third-party, local, separated by blank lines |
| B-STAR-IMPORT | No `from module import *` |
| B-LOGGING | Use `logging.getLogger(__name__)` not `logging.info()` at module level |
| B-ENCODING | All `open()`, `read_text()`, `write_text()`, `subprocess(text=True)` must specify `encoding="utf-8"` on Windows-compatible code |

### Framework-Specific Rules

**Django** (apply only if `django` in imports):

| ID | Rule |
|---|---|
| F-DJ-N1 | Use `select_related()` / `prefetch_related()` for FK access in loops |
| F-DJ-ATOMIC | Multi-step DB writes inside `transaction.atomic()` |

**FastAPI** (apply only if `fastapi` in imports):

| ID | Rule |
|---|---|
| F-FA-CORS | CORS `allow_origins` must be explicit list, not `["*"]` in production |
| F-FA-RESPONSE | All route handlers must declare `response_model` |
| F-FA-BLOCKING | No blocking I/O in `async def` route handlers |

**Flask** (apply only if `flask` in imports):

| ID | Rule |
|---|---|
| F-FL-CSRF | Forms must have CSRF protection |
| F-FL-ERROR | Custom error handlers for 404 and 500 |
