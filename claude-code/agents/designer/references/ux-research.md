# UX Research Reference

UX 리서치 모드에서 사용하는 방법론 및 산출물 가이드.

## 산출물 생성 방식

### designer 도구 설정에 따른 분기

- **HTML/CSS 모드**: 각 UX 산출물을 `mockups/` 디렉토리 아래 HTML 파일로 생성
- **Figma 모드**: 각 UX 산출물을 Figma 캔버스에 시각화 (ui-figma.md 참조)

---

## 유저 페르소나 (User Persona)

### 목적
타겟 사용자를 구체화하여 이후 UI 설계 결정의 기준으로 삼는다.

### 포함 내용
- **이름**: 가상의 대표 사용자 이름
- **나이**: 연령대
- **역할/직업**: 직책 또는 역할
- **목표**: 제품을 사용하는 핵심 목표 (2~3개)
- **페인포인트**: 현재 겪고 있는 불편함 (2~3개)
- **행동 패턴**: 디지털 기기 사용 습관, 접근 방식
- **기술 수준**: 상/중/하

### HTML/CSS 산출물
- 파일 경로: `mockups/persona-{name}.html`
- 카드 레이아웃: 320x400px, 프로필 영역 + 속성 리스트
- 색상: 배경 #F8FAFC, 강조 Primary 색상

### Figma 산출물
- 페르소나 카드: 320x400px Frame
- 별도 Figma 페이지 또는 섹션에 배치

## 유저 저니맵 (User Journey Map)

### 목적
사용자가 제품을 경험하는 전 단계를 시각화하여 개선 포인트를 발견한다.

### 단계 구조
인지 → 탐색 → 결정 → 사용 → 재방문

### 각 단계별 포함 내용
- **행동 (Actions)**: 사용자가 하는 행동
- **생각 (Thoughts)**: 사용자의 내면 생각
- **감정 (Emotions)**: 감정 상태 (이모지 또는 감정 곡선)
- **터치포인트 (Touchpoints)**: 사용자가 접촉하는 채널/기능
- **페인포인트 (Pain Points)**: 불편하거나 막히는 지점

### HTML/CSS 산출물
- 파일 경로: `mockups/journey-map.html`
- 수평 타임라인 레이아웃 (각 단계 카드 너비 균등 분할)
- 프레임 기준: 전체 너비 100%, 최소 높이 400px

### Figma 산출물
- 프레임 크기: 1920x600px
- 수평 타임라인, 각 단계는 세로 구성

## 정보 구조 (Information Architecture, IA)

### 목적
콘텐츠와 기능의 계층 구조를 설계하여 네비게이션 패턴을 결정한다.

### 포함 내용
- **사이트맵**: 최상위 카테고리 → 하위 페이지 트리
- **네비게이션 패턴**: Top Nav, Side Nav, Tab Bar, Breadcrumb 등
- **콘텐츠 계층**: 각 페이지의 주요 콘텐츠 블록 구성
- **주석**: 접근성, 권한, 조건부 노출 등 특이사항

### 표현 방식
- 트리 구조: 노드(사각형) + 연결선
- 상위 노드 → 하위 노드 방향 화살표
- 네비게이션 패턴은 노드 옆 주석으로 표시

### HTML/CSS 산출물
- 파일 경로: `mockups/ia-sitemap.html`
- SVG 또는 CSS Grid로 트리 렌더링

### Figma 산출물
- 별도 페이지 "IA / Sitemap"에 배치
- 연결선: Figma Connector 도구 또는 화살표

## 유저 플로우 (User Flow)

### 목적
특정 태스크 수행 시 사용자의 화면 이동 경로와 분기점을 정의한다.

### 노드 유형
| 노드 | 형태 | 의미 |
|------|------|------|
| 시작/끝 | 원 | 플로우 시작점 또는 종료점 |
| 화면 | 사각형 | 특정 페이지 또는 화면 |
| 분기 | 다이아몬드 | 조건 판단 (예: 로그인 여부) |
| 액션 | 화살표 | 사용자 행동 또는 시스템 반응 |

### 포함 내용
- 시작 조건 (트리거)
- 각 화면에서의 사용자 행동
- 분기 조건 (Yes/No 또는 A/B)
- 오류 경로 및 엣지 케이스
- 종료 조건

### HTML/CSS 산출물
- 파일 경로: `mockups/user-flow-{task-name}.html`
- SVG 플로우 다이어그램 또는 CSS Flexbox 노드 배치

### Figma 산출물
- 별도 섹션 "User Flow"에 배치
- Figma Connector 도구로 화살표 연결

## 와이어프레임 (Wireframe)

### 목적
색상, 폰트, 이미지 없이 레이아웃과 구조만 검증하는 로우파이 목업.

### 디자인 원칙
- **그레이스케일만 사용**: #F3F4F6 배경, #6B7280 요소, #D1D5DB 플레이스홀더
- **실제 콘텐츠 없음**: 텍스트는 Lorem Ipsum 또는 [제목], [버튼] 레이블
- **구조 집중**: 레이아웃, 요소 크기, 여백, 계층 구조
- **하이파이 없음**: 그라디언트, 그림자, 커스텀 색상 금지

### 포함 요소
- 헤더/푸터 영역
- 네비게이션 위치 및 구조
- 주요 콘텐츠 블록 위치
- 인터랙션 요소 (버튼, 폼, 링크) 위치
- 여백 및 간격 표시

### HTML/CSS 산출물
- 파일 경로: `mockups/wireframe-{screen-name}.html`
- 그레이스케일 CSS만 사용
- 사용자 승인 후 하이파이 변환 진행

### Figma 산출물
- 별도 페이지 "Wireframes"에 배치
- 그레이스케일 색상 제한
- 사용자 승인 후 하이파이 페이지로 복사하여 고도화

## UX 작업 순서

복합 UX 작업 시 아래 순서로 진행:

1. **유저 페르소나** 정의 → 타겟 사용자 확정
2. **유저 저니맵** 작성 → 전체 경험 흐름 파악
3. **IA (정보 구조)** 설계 → 콘텐츠/기능 계층 결정
4. **유저 플로우** 정의 → 주요 태스크 경로 확정
5. **와이어프레임** 생성 → 레이아웃 구조 검증
6. 사용자 승인 후 → 하이파이 UI 디자인 (ui-figma.md 또는 mockup-html.md로 전환)
