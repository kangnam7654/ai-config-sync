## 3. 조항별 위반 분석

### 3.1 제15조 (개인정보의 수집-이용) -- 위반

**문제:** 동기화 스크립트(`sync-timestamps.py`)가 `~/.claude/` 및 `~/.openclaw/workspace/` 하위 파일을 자동으로 수집하여 git 저장소에 복사한다. 이 과정에서 개인정보가 포함된 파일(USER.md, memory/*.md, cal_sync.py 등)이 동기화 대상에 포함되나, 개인정보 수집 목적, 항목, 보유기간에 대한 고지가 없다.

**근거:** 프로젝트 어디에도 개인정보 처리 목적, 수집 항목, 보유-이용 기간, 제3자 제공 여부에 대한 고지문이 존재하지 않는다.

### 3.2 제17조 (개인정보의 제공) -- 심각한 위반

**문제:** 저장소가 **public**으로 설정되어 있어, 동기화된 모든 개인정보가 GitHub를 통해 전 세계 불특정 다수에게 제공되고 있다. 이는 정보주체(본인 및 제3자)의 동의 없는 개인정보 제3자 제공에 해당한다.

**특히 심각한 사항:**
- 제3자(현정화 과장)의 실명, 직함, 휴대전화번호가 동의 없이 공개 저장소에 노출
- git 이력(793개 커밋)에 걸쳐 개인정보가 누적되어 있어, 현재 파일 삭제만으로는 이력에서 제거 불가

### 3.3 제21조 (개인정보의 파기) -- 위반

**문제:** 개인정보의 보유기간 또는 처리 목적 달성 후 파기 절차가 정의되어 있지 않다. git 이력에 개인정보가 영구 보존되고 있다. `timestamps/*.json`에 수천 개의 파일 경로와 타임스탬프가 누적되나 파기 정책이 없다.

### 3.4 제29조 (안전성 확보 조치) -- 심각한 위반

**문제:** 개인정보의 안전한 관리를 위한 기술적-관리적 보호 조치가 불충분하다.

구체적 위반 사항:

| 안전성 조치 항목 | 현재 상태 | 평가 |
|---|---|---|
| 접근 권한 관리 | 저장소 public, 접근 제한 없음 | 부적합 |
| 개인정보 암호화 | 평문 저장, 암호화 없음 | 부적합 |
| 접근 기록 관리 | git log만 존재, 체계적 감사 로그 없음 | 미흡 |
| 개인정보 전송 암호화 | git SSH/HTTPS 사용 (전송 구간은 암호화) | 적합 |
| 백업/복구 | 없음 (git 이력 의존) | 미흡 |

### 3.5 제30조 (개인정보 처리방침) -- 위반

**문제:** 개인정보 처리방침이 존재하지 않는다. README.md에 "민감 정보는 템플릿/로컬 파일로 분리해 관리합니다"라는 언급이 있으나, 이는 개인정보 처리방침의 법적 요건을 충족하지 않는다.

### 3.6 제59조 (금지행위) -- 잠재적 위반

**문제:** `openclaw/workspace/sync-openclaw.sh:42`에서 `gh auth token` 결과를 URL에 직접 삽입하는 패턴이 존재한다. 이 스크립트가 동기화 대상에 포함되어 있어, 토큰 값이 git 이력에 기록될 위험이 있다.

## 8. 결론

본 프로젝트의 핵심 목적(개인 기기 간 설정 동기화)은 정당하나, **공개 저장소에 개인정보가 포함된 파일이 동기화되고 있다는 점이 가장 심각한 문제**이다. 특히 제3자(현정화 과장)의 실명과 휴대전화번호가 공개 저장소의 git 이력에 포함되어 있어, 개인정보보호법 제17조(제3자 제공) 위반에 해당하며 해당 정보주체에 대한 유출 통지 의무(제34조)가 발생할 수 있다.

**최우선 조치:** 저장소를 즉시 private으로 전환하고, 제3자 개인정보를 git 이력에서 완전히 제거하라.
