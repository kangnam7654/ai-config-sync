### 3.2 제17조 (개인정보의 제공)

**판정: 부적합 (위반)**

## 5. 기존 보안 조치 인정 사항

프로젝트에 이미 적용된 긍정적 보안 관행:

1. **민감 설정 파일 분리**: `openclaw.json`(토큰 포함)은 `.gitignore`로 추적 제외. 템플릿(`openclaw.template.json`)만 관리.
2. **자격증명 디렉토리 제외**: `.gitignore`에 `credentials/`, `auth-profiles.json` 명시.
3. **화이트리스트 방식**: `CLAUDE_INCLUDES`로 claude-code 동기화 대상을 명시적으로 제한.
4. **제외 패턴**: `EXCLUDES` 딕셔너리로 `history.jsonl`, `telemetry`, `sessions` 등 제외.
5. **경로 검증**: `validate_filepath()`로 path traversal 방지.
6. **타임스탬프 검증**: `validate_timestamp()`로 미래/음수 값 차단.
7. **환경변수 사용 (부분)**: `NOTION_API_KEY`는 환경변수로 관리.
8. **SSH 전송**: Git remote가 SSH(`git@github.com:`) 프로토콜 사용.

## 8. 결론

이 프로젝트의 가장 심각한 문제는 **PUBLIC GitHub 저장소에 다수의 개인정보가 평문으로 노출**되어 있다는 점이다. 특히:

1. **타인(현정화 과장)의 실명 + 휴대폰 번호**가 동의 없이 공개되어 있어 개인정보보호법 제17조(동의 없는 제3자 제공) 위반에 해당하며, 제71조에 따라 5년 이하 징역 또는 5천만원 이하 벌금 대상이 될 수 있다.

2. 본인의 **실명, 이메일, Telegram ID, 연봉, 거주지 상세 주소(동호수 포함), 건강정보(코로나 확진), 대출 정보, 경력 이력**이 전 세계 누구나 접근 가능한 상태로 공개되어 있어, 스토킹/피싱/사회공학 공격의 직접적 타겟이 될 수 있다.

3. 동기화 목적("설정 파일 동기화")에 비해 수집/공개되는 개인정보의 범위가 과도하게 넓다. `workspace` 섹션이 메모리 파일, 캘린더 동기화 스크립트, 개인 일정 데이터까지 포함하고 있어 최소 수집 원칙에 위배된다.

**즉시 조치가 필요한 사항**: (1) 타인 개인정보 삭제 및 git 이력 제거, (2) 저장소 PRIVATE 전환, (3) 동기화 범위에서 개인정보 포함 파일 제외.
