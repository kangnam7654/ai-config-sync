# Idea Brief: TodoFlow

## 프로젝트 개요

- **앱명**: TodoFlow
- **한 문장 요약**: React + FastAPI + SQLite 기반 미니멀 할일 관리 웹앱
- **타겟 사용자**: 풀스택 기술 학습이 필요한 개발자, 개인 생산성 관리 사용자
- **핵심 기능**:
  1. 할일 추가 -- 제목 입력으로 새 할일 생성
  2. 할일 삭제 -- 개별 할일 영구 삭제
  3. 할일 완료 -- 완료/미완료 토글
- **차별점**: React+FastAPI+SQLite 풀스택의 최소 기능 완성형(MVC) 아키텍처 레퍼런스

## 시장 분석

- **TAM**: 글로벌 생산성 앱 시장 $96.36B (2025)
- **SAM**: 개인 할일 관리 앱 $4.8B
- **SOM**: 한국 개인 사용자 $12M
- **경쟁사**: Todoist (크로스플랫폼, 복잡), Microsoft To Do (MS 종속), TickTick (프리미엄 잠금)
- **기회**: 풀스택 학습용 레퍼런스 앱 수요 지속. React+FastAPI 조합 레퍼런스 부족.
- **리스크**: 상용 앱과 기능 경쟁 불가. 레퍼런스 목적 범위 이탈 시 가치 상실.

## 전략 검증 결과

- **CSO 판정**: Go
- **시장 진입 타이밍**: 적절 (풀스택 레퍼런스 수요 지속)
- **경쟁 우위 지속 가능성**: 상용 제품과 비경쟁. 교육/레퍼런스 카테고리 포지셔닝.
- **리스크 대응**:
  - 기능 부족 → 레퍼런스 목적 명확화
  - 유사 프로젝트 → 코드 품질로 차별화
  - 유지보수 → 최소 기능으로 부담 최소화
