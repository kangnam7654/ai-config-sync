# Business Model: 달결 (DalGyeol)

## 1. Product Overview

### 제품 설명
달결은 사주(四柱) 기반 운세 분석 및 AI 상담 서비스를 제공하는 iOS 네이티브 앱이다. 순수 Rust 규칙 기반 사주 엔진(비용 $0)으로 운세를 생성하고, 멀티 LLM 프로바이더를 활용한 AI 상담사와 실시간 채팅 상담을 제공한다.

### 핵심 가치
| 가치 | 설명 |
|------|------|
| 불확실성 해소 | "이 선택이 맞을까?"에 대한 사주 기반 답변 |
| 개인 맞춤 해석 | 생년월일시 기반 정밀 사주 분석 |
| AI 즉시 상담 | 24시간 AI 상담사와 사주 맥락 기반 실시간 대화 |
| 프라이버시 | 생년월일 등 민감 정보 AES-GCM 암호화 저장 |
| 비용 효율성 | 규칙 기반 엔진으로 서버당 수만 건 처리 가능 (API 비용 $0) |

### 타겟 고객
| 세그먼트 | 특성 | 주요 니즈 | 비중 |
|---------|------|----------|------|
| 코어 (20~30대 여성) | 연애/직장 고민, 점술 콘텐츠에 익숙 | 불확실성 해소, 위로, 맞춤 해석 | 50% |
| 확장 (30~40대 남녀) | 이직/재물/건강 관심 | 의사결정 보조, 심리적 확신 | 30% |
| 캐주얼 (전 연령) | SNS 공유, 가벼운 호기심 | 재미, 화제거리, 궁합 | 20% |

### 현재 상태
- 사용자: 약 500명 (iOS only, 한국 시장)
- 매출: 0원 (무료 운영 중)
- 투자: 부트스트랩 (자체 운영)
- 기술 인프라: Rust 백엔드(Axum) + SwiftUI iOS 앱 + PostgreSQL + 포인트 시스템/IAP 구현 완료

## 3. Revenue Model

### 선택 모델: 하이브리드 (포인트 IAP + 보상형 광고 + 구독)

| 근거 | 설명 |
|------|------|
| 제품 특성 적합 | 사주/운세 앱은 사용 빈도 불규칙. 구독만으로는 비정기 사용자 이탈, 종량제만으로는 습관 유저 수익화 실패 |
| 경쟁사 패턴 일치 | 상위 앱(포스텔러, 점신) 모두 하이브리드 모델 채택 |
| 비용 구조 최적 | 규칙 기반 엔진 비용 $0 → 무료 제공 부담 최소 → 프리미엄 전환에 집중 |
| 이미 구축 완료 | 포인트 시스템, IAP, 광고 보상 인프라 이미 구현 → 추가 개발 불필요 |

### 수익원 비중 목표 (12개월 후)

| 수익원 | 비중 | 역할 |
|--------|------|------|
| 포인트팩 IAP | 50% | 핵심 매출 (비정기 구매자) |
| 구독 | 25% | 반복 매출 (MRR) |
| 보상형 광고 | 15% | 무료 유저 수익화 |
| 시즌 단품 | 10% | 시즌 매출 (신년운세 등) |

### 기각한 대안

| 대안 | 기각 이유 |
|------|----------|
| 구독 전용 | 비정기 사용자가 많아 구독 피로도 높음. 포스텔러 구독 인상 후 전환율 하락 사례 |
| 광고 전용 | 500 DAU 기준 일 500~1,500원 수준. 서버비(월 5만원)도 미달 |
| 건당 직접 과금 | 포인트/코인이 지불 고통 완화 효과 입증됨. 직접 원화 결제보다 전환율 높음 |

### 가치 지표

| 가치 지표 | 과금 단위 | 핵심 |
|----------|----------|------|
| 운세 해석의 깊이 | 포인트 (유형별 차등) | "더 자세히 보려면 포인트" |
| AI 상담 질문 수 | 포인트 (질문당 10P) | "질문할수록 포인트 소비" |
| 매일 콘텐츠 | 구독 월정액 | "매일 상세 운세 + 포인트" |
| 광고 없는 경험 | 구독 부가 혜택 | 부가 가치 |

## 4. Pricing Strategy

### 가격 포지셔닝: 저가 침투 전략

시장 대비 30~50% 저렴하게 진입. 500명 규모에서 전환 데이터 확보 후 단계적 인상.

### 무료/유료 경계 논리

**무료 (가치 체험 충분히):**
- 오늘의 운세 요약 (매일 새로움 → 습관 형성)
- 사주팔자 기본 분석 (핵심 가치 1회 체험)
- 운세 캘린더 월별 점수 (DAU 드라이버)
- 출석 체크 포인트 (5P/일, 7일 보너스 +20P)
- 광고 시청 포인트 (5P x 5회/일)
- 가입 보너스 100P (유료 기능 1~2회 체험 가능)
- AI 상담 첫 질문 (AI 가치 경험)

**유료 (더 원하게 만드는 지점):**
- 더 **자세한** 해석 → 상세 운세, 종합 리포트
- 더 **개인화된** 답변 → AI 상담 추가 질문
- 더 **넓은 범위** → 주간/월간/궁합
- **특정 주제** 전문 리포트 → 연애운, 재물운

**경계 설정 근거:**
- 가입 보너스 100P로 유료 기능 2~3회 체험 가능 (상세 운세 15P x 2 + AI 질문 10P x 7)
- 무료 포인트 수급량: 출석 5P + 광고 25P = 30P/일 → 상세 운세 2일에 1회 가능
- 이 속도로는 종합 리포트(60P)를 보려면 2일 이상 필요 → 충전 동기 발생
- "맛보기는 충분하지만, 더 원하면 결제" 지점

### 경쟁사 대비 포지션

```
가격 ↑
     |  정통만세력 (전문가 고가)
     |  포스텔러 (다양한 콘텐츠, 중~고가)
     |  신한운세 (전통, 중가)
     |
     |  ★ 달결 (AI 차별화, 저~중가)  ← 진입 포지션
     |
     |  운세의 신 (캐주얼, 저가)
     |  점신 (광고 기반, 무료~저가)
가격 ↓
```

## 6. Risk Assessment

### 상위 3 리스크 + 대응 방안

#### 리스크 1: 전환율 미달 (심각도: 높음, 확률: 40%)

| 항목 | 내용 |
|------|------|
| 시나리오 | 가입 보너스 100P 소진 후 추가 결제 없이 이탈. 무료 콘텐츠만 소비 |
| 근본 원인 | 무료→유료 전환 트리거 부족, AI 답변 품질 미흡, 유료 가치 체감 부족 |
| 즉시 대응 | 100P 소진 시점에 "첫 구매 30% 할인" 팝업. 유료 운세 미리보기(일부만 공개) |
| 단기 대응 | AI 상담 프롬프트 최적화, Firebase Analytics 전환 퍼널 추적 |
| 중기 대응 | A/B 테스트 인프라, 가격 실험, 전환 트리거 다양화 |

#### 리스크 2: 리텐션 저조 (심각도: 높음, 확률: 35%)

| 항목 | 내용 |
|------|------|
| 시나리오 | 일일 운세 + 사주 한 번 확인 후 재방문 없음. D7 리텐션 10% 이하 |
| 근본 원인 | 매일 볼 콘텐츠 부족, 푸시 알림 미구현, 습관 루프 미형성 |
| 즉시 대응 | 출석 체크 + 광고 포인트 적극 노출 (이미 구현) |
