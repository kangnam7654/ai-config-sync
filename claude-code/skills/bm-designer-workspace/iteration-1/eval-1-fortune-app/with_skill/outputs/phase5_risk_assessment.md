# Phase 5: 전략 검증 (리스크 분석)

## 검증 항목별 분석

### 1. 수익 모델의 지속 가능성

#### 경쟁사 대응 시나리오

| 시나리오 | 확률 | 영향 | 대응 |
|---------|------|------|------|
| 점신이 AI 상담 추가 | 높음 (60%) | 중간 | 달결의 사주 엔진 정확도 + AI 상담 품질 차별화. 점신은 광고 모델이므로 유료 상담에 집중도 낮을 가능성 |
| 포스텔러가 가격 인하 | 중간 (40%) | 높음 | 이미 저가 포지셔닝이므로 영향 제한적. 포인트 시스템의 유연성으로 프로모션 대응 |
| 신규 AI 운세 앱 진입 | 높음 (70%) | 중간 | 선점 효과 + 규칙 기반 엔진의 비용 우위(비용 $0). AI만 의존하는 앱은 API 비용 부담 |

#### 시장 변화 시나리오

| 시나리오 | 확률 | 영향 | 대응 |
|---------|------|------|------|
| AI 운세에 대한 규제 강화 | 낮음 (10%) | 높음 | 사주 엔진은 규칙 기반이므로 AI 규제 영향 제한적. 면책 문구 강화 |
| 운세 앱 시장 축소 | 낮음 (5%) | 높음 | 한국 점술 시장은 문화적으로 견고. 최근 MZ세대 유입 증가 추세 |
| Apple 수수료 인상 | 낮음 (10%) | 중간 | Gross Margin 82~85%이므로 5~10%p 인상에도 흑자 유지 가능 |

**판정: 관리 가능한 리스크** - 모델 자체를 흔드는 치명적 리스크 없음.

### 2. 가격 전략의 적정성

#### 고객 지불 의향 대비 분석

| 상품 | 달결 가격 | 시장 평균 | 지불 의향 대비 | 판정 |
|------|----------|----------|-------------|------|
| 상세 운세 | 585원 (15P) | 1,500원 | 아래 | 저가 - 전환율 높이기 적합 |
| 궁합 | 1,365원 (35P) | 7,900원 | 크게 아래 | 업셀링 여지 큼 |
| 종합 리포트 | 2,340원 (60P) | 12,450원 | 크게 아래 | 가격 인상 가능 |
| AI 상담 | 390원/질문 | 1,000원/질문 | 아래 | 적절 |

**우려**: 가격이 시장 대비 **지나치게 낮을 수 있음**. 초기 전환 데이터 확보에는 유리하나, LTV가 낮아질 위험.

**권고**:
- 초기 3개월은 현재 저가로 유지하여 전환율 데이터 확보
- 전환율 8% 이상 확인 시 단계적 가격 인상 (15P→20P, 35P→50P)
- A/B 테스트 인프라 구축 후 최적 가격 탐색

### 3. 유닛 이코노믹스의 현실성

#### 가정 검증

| 가정 | 값 | 낙관적? | 검증 방법 |
|------|-----|---------|----------|
| 전환율 6% | 업종 평균 5~8% 중간 | 적정 | 출시 후 30일 데이터 |
| ARPPU 10,000원 | 업종 평균 8,000~12,000원 중간 | 적정 | 포인트팩 구매 패턴 추적 |
| 월 성장 15% | 마케팅 없이 ASO+바이럴 | **다소 낙관적** | 3개월 추이 확인 필요 |
| Churn 18% | 업종 평균 15~20% | 적정 | D30 리텐션 추적 |
| 광고 eCPM 1,500원 | 한국 리워드 광고 평균 | 적정 | AdMob 대시보드 |

**우려**: 월 성장 15%는 마케팅 없이 달성하기 쉽지 않음. 바이럴 기능(궁합 공유) 미구현 상태에서 10% 이하가 현실적.

**권고**: 보수적 시나리오(월 성장 5%)를 기준으로 계획하고, 궁합 공유 기능 출시 후 성장률 재평가.

### 4. 스케일링 시 비용 구조 변화

#### 규모의 경제 분석

| 유저 규모 | 서버비 | AI API 비용 | Apple 수수료 | 총 비용 | Gross Margin |
|----------|--------|------------|-------------|--------|-------------|
| 500명 | 50,000원 | ~15,000원 | 매출 15% | ~71,750원 | 82% |
| 2,000명 | 100,000원 | ~60,000원 | 매출 15% | ~220,000원 | 83% |
| 10,000명 | 200,000원 | ~300,000원 | 매출 15% | ~900,000원 | 84% |
| 50,000명 | 500,000원 | ~1,500,000원 | 매출 15% | ~4,000,000원 | 85% |

**규모의 경제 존재**:
- 서버비(고정비)는 유저당 비용이 급감 (500명: 100원/인 → 50,000명: 10원/인)
- AI API 비용(변동비)은 비례 증가하나, 유료 유저 비율에 따라 제한적
- 사주 엔진이 규칙 기반(비용 $0)이므로 무료 유저 증가가 비용 부담 없음
- Apple 수수료는 비율 고정(15%)이므로 마진 영향 없음

**병목 지점**:
- 10,000명 이상에서 AI API 비용이 주요 변동비로 부상
- LLM 프로바이더 협상력 확보 또는 자체 모델 검토 시점

### 5. 최대 리스크 3가지 + 대응 방안

#### 리스크 1: 전환율 미달 (500명 중 유료 유저 확보 실패)

| 항목 | 내용 |
|------|------|
| **심각도** | 높음 |
| **확률** | 중간 (40%) |
| **시나리오** | 가입 보너스 100P 소진 후 추가 결제 없이 이탈. 무료 콘텐츠만 소비 |
| **근본 원인** | 무료→유료 전환 트리거 부족, AI 답변 품질 불충분, 가치 체감 부족 |
| **대응 방안** | |
| 즉시 | 가입 보너스 100P가 소진되는 시점에 "30% 할인 첫 구매 팝업" 노출 |
| 단기 | AI 상담 첫 경험 품질 향상 (프롬프트 최적화), 유료 운세 미리보기(일부 공개) |
| 중기 | 전환 퍼널 분석 도구 도입 (Firebase Analytics), A/B 테스트 인프라 |

#### 리스크 2: 리텐션 저조 (DAU 유지 실패)

| 항목 | 내용 |
|------|------|
| **심각도** | 높음 |
| **확률** | 중간 (35%) |
| **시나리오** | 일일 운세 + 사주팔자 한 번 확인 후 재방문 없음. D7 리텐션 10% 이하 |
| **근본 원인** | 매일 볼 콘텐츠 부족, 푸시 알림 부재, 습관 루프 미형성 |
| **대응 방안** | |
| 즉시 | 출석 체크 보상 강화 (이미 구현), 매일 다른 운세 키워드 제공 |
| 단기 | 푸시 알림 도입 (매일 오전 "오늘의 운세가 도착했어요"), 운세 캘린더 적극 노출 |
| 중기 | 시간대별 운세(12시진) 추가로 앱 열기 빈도 증가, 커뮤니티 기능 검토 |

#### 리스크 3: 바이럴 성장 실패 (유저 증가 정체)

| 항목 | 내용 |
|------|------|
| **심각도** | 중간 |
| **확률** | 높음 (50%) |
| **시나리오** | 마케팅 예산 0원, 오가닉 성장 월 2~3%에 불과. 12개월 후에도 600~700명 |
| **근본 원인** | 바이럴 기능(궁합 공유) 미구현, ASO 최적화 부족, 입소문 트리거 없음 |
| **대응 방안** | |
| 즉시 | ASO 최적화 (키워드: 사주, 운세, AI상담, 궁합). 앱스토어 스크린샷 개선 |
| 단기 | 궁합 기능 우선 개발 + 공유 카드 이미지 생성. 카카오톡/인스타 공유 동선 |
| 중기 | 레퍼럴 프로그램 (친구 초대 시 50P), 인플루언서 무료 체험 배포 |
| 장기 | 소규모 마케팅 예산 투입 (월 10~20만원 인스타/틱톡), CPI 측정 후 확대 |

## 종합 판정

| 검증 항목 | 판정 | 요약 |
|----------|------|------|
| 수익 모델 지속 가능성 | 관리 가능 | 규칙 기반 엔진의 비용 $0이 핵심 경쟁력. 경쟁사 대응 가능 |
| 가격 전략 적정성 | 경미한 우려 | 초기 저가 전략 적합하나, LTV 극대화를 위해 단계적 인상 필요 |
| 유닛 이코노믹스 현실성 | 관리 가능 | 월 성장 15% 가정이 다소 낙관적. 보수적 시나리오에서도 7개월 내 BEP |
| 스케일링 비용 구조 | 양호 | 규모의 경제 명확. AI API가 유일한 변동비 부담 요인 |
| 치명적 리스크 | 없음 | 전환율 미달이 가장 큰 리스크이나 대응 방안 존재 |

**결론**: Phase 3로 돌아가 모델을 수정할 필요 없음. 리스크 대응 방안을 BM 문서에 포함하고 모니터링 체계를 수립.
