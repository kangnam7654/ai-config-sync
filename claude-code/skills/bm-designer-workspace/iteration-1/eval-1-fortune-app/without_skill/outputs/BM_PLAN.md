# 달결 사주 운세 앱 - 수익화 BM 설계서

> 현재 상태: 무료 운영, MAU 500명, 월 서버비 5만원, AI API 건당 50원, iOS 한국 시장 전용
> 목표: 6개월 내 BEP(손익분기점) 달성, 12개월 내 월 매출 300만원 도달

## 1. 현황 분석

### 1.1 비용 구조

| 항목 | 월 비용 | 비고 |
|------|---------|------|
| 서버 (Fly.dev) | 50,000원 | 고정비 |
| AI API 비용 | 변동 | 건당 50원, 현재 무료라 추정 어려움 |
| Apple Developer Program | 약 11,000원 | 연 $99 / 12개월 |
| **현재 총 고정비** | **약 61,000원/월** | AI 비용 제외 |

### 1.2 AI API 비용 시뮬레이션

| 시나리오 | 일 평균 AI 호출 | 월 호출 수 | 월 AI 비용 |
|----------|---------------|-----------|-----------|
| 현재 (500 MAU, 무료) | DAU 100 x 2회 | 6,000건 | 300,000원 |
| 유료 전환 후 | DAU 100 x 0.5회 | 1,500건 | 75,000원 |

**핵심 인사이트**: AI API를 무제한 무료로 제공하면 MAU 500에서도 월 30만원 손실이 발생한다. 규칙 기반 엔진(비용 0원)과 AI 엔진(건당 50원)의 명확한 경계가 수익화의 전제 조건이다.

### 1.3 경쟁 환경 (한국 시장)

| 앱 | 추정 매출 | BM 핵심 |
|----|----------|---------|
| 점신 | ~830억/년 | 광고 + 다양한 콘텐츠(관상, 꿈, 사주 등) |
| 포스텔러 | ~100억+/년 | 46+ 테마 유료 리포트, 프리미엄 해석 |
| 정통사주 | 소규모 | 대운/세운 전문 분석 |
| 신한사주 | 소규모 | 커뮤니티 + 전문 상담 |

**시사점**: 상위 앱은 "무료 콘텐츠로 습관 형성 -> 유료 프리미엄 해석"이 공통 패턴이다. 500명 규모에서는 광고 수익은 미미하므로(eCPM 1,000원 기준 월 3,000원 수준) 직접 결제 전환이 유일한 현실적 수익원이다.

## 2. 핵심 전략: "규칙 기반 무료 + AI 프리미엄 유료"

### 2.1 전략 요약

```
무료 콘텐츠 (비용 $0)         유료 콘텐츠 (포인트 차감)
 규칙 기반 사주 엔진            AI LLM 기반 심층 해석
 ========================      ========================
 오늘의 운세 요약               오늘의 상세 운세 (15P)
 사주팔자 기본                  종합 사주 리포트 (60P)
 운세 캘린더                    주간 운세 (25P)
 원카드 타로 프리뷰             월간 운세 (40P)
                               궁합 분석 (35P)
                               AI 상담 질문 (10P/회)
                               타로 전체 해석 (15P)
```

**비용 구조의 핵심**: 규칙 기반 엔진은 서버 연산만으로 동작하므로 추가 비용이 0원이다. AI LLM 호출은 건당 50원이 발생하므로 반드시 유료화한다. 이 경계선이 BM의 근간이다.

### 2.2 왜 이 전략인가

| 고려한 대안 | 기각 이유 |
|------------|----------|
| 전면 구독제 (월정액만) | MAU 500에서 구독 전환율 3% 가정 시 15명, 월 10만원. 비정기 사용자가 많은 운세 앱에 부적합 |
| 광고 수익 중심 | MAU 500으로는 월 수천원 수준. 최소 MAU 5만 이상이어야 유의미 |
| 전문 상담사 연결 | 수수료 모델은 양면 마켓 구축이 필요. 현 단계에 과도한 복잡성 |
| **포인트 과금 + 구독 보조** | **채택. 비정기 소비 패턴에 적합. 소액 결제 장벽 낮음. 구독은 헤비유저 리텐션 장치** |

## 3. 수익 모델 상세

### 3.1 메인 BM: 포인트(크레딧) 결제

사주 앱은 매일 쓰는 유저와 가끔 쓰는 유저가 혼재한다. 구독만으로는 가끔 쓰는 유저를 놓친다. 포인트 차감형이 이 특성에 가장 적합하다.

#### 포인트팩 상품

| 포인트팩 | 가격 (Apple IAP) | 단가 | 포지션 |
|---------|------------------|------|--------|
| 100P | 3,900원 | 39원/P | 첫 결제 진입용 |
| 300P | 8,900원 | ~30원/P | 주력 상품 |
| 700P | 19,900원 | ~28원/P | 헤비유저용, 볼륨 할인 |

**가격 책정 근거**:
- Apple 30% 수수료 후 실 수입: 100P 팩 기준 2,730원
- AI API 비용: 100P로 AI 상담 10회 = 500원
- 100P 팩 순이익: 약 2,230원 (마진율 57%)
- 한국 모바일 콘텐츠 결제 평균 단가 5,000~10,000원 대비 3,900원은 진입 장벽이 낮음

#### 포인트 소비 상품

| 상품 | 소모 포인트 | AI 비용 | 순이익/건 | 엔진 |
|------|-----------|---------|----------|------|
| AI 질문 1회 | 10P (~390원) | 50원 | 223원 | LLM |
| 오늘의 상세 운세 | 15P (~585원) | 50원 | 360원 | LLM |
| 쓰리카드 타로 | 15P (~585원) | 0원 | 410원 | 규칙 기반 |
| 주간 운세 | 25P (~975원) | 50원 | 633원 | LLM |
| 궁합 분석 | 35P (~1,365원) | 0원 | 956원 | 규칙 기반 |
| 월간 운세 | 40P (~1,560원) | 50원 | 1,042원 | LLM |
| 켈틱크로스 타로 | 40P (~1,560원) | 0원 | 1,092원 | 규칙 기반 |
| 종합 사주 리포트 | 60P (~2,340원) | 100원 | 1,538원 | LLM |

> 순이익 계산: (포인트 x 단가 x 0.7 Apple 수수료) - AI 비용. 단가는 300P 팩(30원/P) 기준.

### 3.2 보조 BM: 구독제

포인트만으로는 반복 매출(MRR)을 예측하기 어렵다. 구독은 헤비유저를 고정 매출로 전환하는 장치다.

| 플랜 | 월 가격 | 혜택 | 타겟 |
|------|--------|------|------|
| 라이트 | 6,900원 | 매일 상세 운세 + 월 100P + 광고 제거 | 매일 앱 여는 유저 |
| 프리미엄 | 12,900원 | 월 300P + 리포트 할인 30% + 광고 제거 + 상담 우선 응답 | 적극적 유료 유저 |

**구독 가치 분석**:
- 라이트: 매일 상세 운세를 포인트로 구매 시 15P x 30일 = 450P = 약 13,500원. 6,900원 구독은 49% 할인 효과
- 프리미엄: 300P 팩(8,900원) + 매일 상세 운세(13,500원 상당) = 22,400원 상당. 12,900원 구독은 42% 할인 효과
- Apple 수수료 후: 라이트 4,830원, 프리미엄 9,030원 실 수입

### 3.3 보조 BM: 보상형 광고

MAU 500에서 배너 광고 수익은 무의미하다. 그러나 보상형 광고는 두 가지 전략적 가치가 있다.

1. **무료 유저 리텐션**: 광고를 보면 포인트를 주므로 무료 유저가 계속 앱을 사용한다
2. **유료 전환 퍼널**: 무료 포인트로 유료 콘텐츠를 맛보게 해서 추가 결제를 유도한다

| 보상 유형 | 1회 보상 | 일일 한도 | 일일 최대 |
|----------|---------|----------|----------|
| 직접 포인트 지급 | 5P | 5회 | 25P |
| 출석 체크 | 5P (7일차 +20P 보너스) | 1회 | 5~25P |

**월간 무료 획득량**: 약 900~1,050P. 포인트팩 100P(3,900원) 10개분에 해당하지만, 매일 접속해야 하므로 DAU 유지에 기여한다. 상세 운세 2일에 1회 무료 이용 수준으로, 유료 전환을 저해하지 않는 설계다.

### 3.4 단품 리포트 (시즌 상품)

| 상품 | 시기 | 예상 가격 | 특징 |
|------|------|----------|------|
| 2027 신년운세 리포트 | 12~1월 | 9,900원 (직접 IAP) | 연간 매출 30~40% 집중 시기 |
| 시험운/합격운 | 수능, 공시 시즌 | 5,900원 | 타겟 명확, 키워드 SEO |
| 재회 가능성 분석 | 상시 | 4,900원 | 감정적 니즈 높음 |
| 이직 타이밍 분석 | 상시 | 4,900원 | 30~40대 타겟 |

## 4. 전환 퍼널 설계

### 4.1 유저 여정

```
[앱 설치] → [회원가입] → [프로필 등록] → [무료 운세 경험] → [신뢰 형성] → [첫 결제] → [반복 구매]
   100%        60%           50%            45%              25%         5~8%        30% 재구매
```

### 4.2 Stage 1: 무료 유입 (D0)

가입 직후 즉시 가치를 보여줘야 한다. "맞는지" 느끼기 전에 결제를 요구하면 안 된다.

**무료 제공 콘텐츠**:
- 오늘의 운세 요약 (규칙 기반, 비용 0원)
- 사주팔자 기본 분석 (규칙 기반, 비용 0원)
- 운세 캘린더 월간 점수 (규칙 기반, 비용 0원)
- AI 상담 첫 질문 1회 무료 (비용 50원, 신뢰 형성 투자)
- 원카드 타로 프리뷰 (카드명 + 해석 첫 줄)
- **가입 보너스 100P** (약 3,900원 상당, 유료 콘텐츠 체험용)

**가입 보너스 100P의 전략적 의미**:
- 상세 운세 6~7회 또는 AI 상담 10회 체험 가능
- 유료 콘텐츠의 가치를 직접 경험시켜 첫 결제 저항을 낮춤
- 비용: 유저당 0원 (실제 현금 유출 없음, 기회비용만 존재)

### 4.3 Stage 2: 신뢰 형성 (D1~D7)

```
DAU 유지 장치:
  - 매일 바뀌는 오늘의 운세 (앱을 열 이유)
  - 출석 체크 포인트 적립 (습관 형성)
  - 운세 캘린더 일별 점수 변화 (호기심)
  - 원카드 타로 프리뷰 (매일 다른 카드)

신뢰 형성 장치:
  - AI 상담 첫 질문 무료 (AI 품질 경험)
  - 운세 상세의 일부 공개 (더 보고 싶은 욕구)
  - 사주 기본 분석의 정확도 (생년월일 기반 개인화)
```

### 4.4 Stage 3: 첫 결제 전환 (D7~D14)

가입 보너스 100P를 소진한 시점이 첫 결제 적기다. 결제 저항을 최소화하는 저가 진입 상품이 핵심이다.

| 전환 트리거 | 상품 | 가격 |
|------------|------|------|
| 보너스 포인트 소진 알림 | 100P 팩 | 3,900원 |
| "상세 운세가 더 궁금하다면" | 100P 팩 | 3,900원 |
| StoreKit Introductory Offer | 첫 구매 할인 팩 | 2,900원 (특별가) |

### 4.5 Stage 4: 반복 구매 -> 구독 전환

```
포인트 3회 이상 구매 유저
  → "구독하면 매달 300P + 매일 상세 운세가 포함됩니다"
  → 구독 전환 유도

높은 AI 상담 사용 유저
  → "프리미엄 구독으로 상담 우선 응답을 받으세요"
  → 구독 전환 유도
```

## 6. 비용 최적화 전략

### 6.1 AI API 비용 통제

현재 건당 50원은 관리하지 않으면 매출을 잠식한다. 다음 전략으로 비용을 통제한다.

|------|------|----------|

| 응답 캐싱 | 동일 input_hash(사용자+날짜+타입)에 대한 결과 재사용 | AI 호출 20% 감소 |

| LLM 계층화 | 일반 질문: 저가 모델(Haiku), 심층 분석: 고급 모델 | 평균 건당 비용 30원으로 감소 |
| 무료 유저 차단 | AI 기능은 포인트 소비 필수 (첫 질문 1회만 무료) | 무과금 호출 제거 |

## 7. 무료 vs 유료 경계선

이 경계선이 BM의 성패를 결정한다. 무료가 너무 빈약하면 이탈하고, 너무 풍부하면 결제하지 않는다.

### 7.1 원칙

```
무료 = "맞는 것 같다"는 신뢰를 형성하는 데 충분한 양
유료 = "더 자세히, 더 깊이, 더 개인적으로" 알고 싶을 때
```

### 7.2 구체적 경계

| 콘텐츠 | 무료 범위 | 유료 범위 | 전환 트리거 |
|--------|----------|----------|------------|
| 오늘의 운세 | 종합 점수 + 한 줄 조언 + 행운 키워드 | 카테고리별 상세 해석 + AI 보강 | "더 자세한 운세가 궁금하다면" |
| 사주팔자 | 4기둥 + 오행 밸런스 + 기본 성격 | 십신 상세 + 대운 + AI 종합 리포트 | "당신의 사주를 AI가 심층 분석합니다" |
| 타로 | 원카드 프리뷰 (카드명 + 첫 줄) | 전체 해석 + 사주 연동 + 쓰리카드/켈틱크로스 | 블러 처리된 나머지 해석 |
| 궁합 | 종합 점수만 | 카테고리별 상세 + 조언 | "두 분의 궁합을 자세히 알아보세요" |
| AI 상담 | 첫 질문 1회 무료 | 이후 질문당 10P | "더 물어보고 싶다면" |
| 운세 캘린더 | 이번 달 전체 | - | (DAU 드라이버, 무료 유지) |
| 출석 체크 | 매일 5P | - | (리텐션 장치, 무료 유지) |

### 7.3 "맛보기" 전략의 구체적 구현

**타로 프리미엄 프리뷰 모델** (가장 효과적인 전환 장치):

```
홈 화면에 매일 자동 노출
  ┌──────────────────────────────┐
  │  오늘의 타로                    │
  │  ┌────────────────────────┐  │
  │  │  카드 이미지 (블러 처리)  │  │
  │  │  운명의 수레바퀴           │  │
  │  └────────────────────────┘  │
  │                              │
  │  "변화의 바람이 불어오고..."   │  <- 첫 줄만 공개
  │  ########################### │  <- 나머지 블러 처리
  │                              │
  │  [전체 해석 보기 - 15P]       │  <- CTA
  └──────────────────────────────┘
```

- 매일 다른 카드가 나와 호기심을 유발한다
- 카드 이름과 첫 줄만 보여주고 나머지를 블러 처리하여 "더 보고 싶은" 욕구를 자극한다
- 15P(약 585원)로 낮은 결제 장벽

## 8. 가격 정책 원칙

### 8.1 5대 원칙

1. **무료 경험이 충분해야 한다** - "맞는지" 느끼기 전에 결제를 유도하면 이탈한다
2. **첫 결제 장벽을 낮춰야 한다** - 3,900원(100P)이 최저 진입점. 향후 2,900원 할인 팩 검토
3. **가격 구조를 단순하게 유지한다** - 포인트 1개 체계. 복잡한 패키지/번들은 결제를 포기하게 만든다
4. **포인트 메인 + 구독 보조** - 비정기 사용자가 많으므로 포인트가 기본, 헤비유저만 구독
5. **초기에는 전환 데이터가 가격보다 중요하다** - 단순하게 시작하고, 데이터를 보고 A/B 테스트

### 8.2 포인트 가치 설계

```
1 포인트 = 약 30~39원 (구매 팩에 따라 변동)

사용자 심리:
  "10P로 질문 하나" = "약 300~400원" = 커피 한 잔의 1/10
  "60P로 종합 리포트" = "약 1,800~2,340원" = 카페 음료 1잔

목표: "이 가격이면 해볼 만하다"는 인식
```

### 8.3 Apple IAP 가격 티어

Apple의 한국 가격 티어에 맞춘 설정:
