# Business Model: HR SaaS (B2B 인사관리 솔루션)

## 1. Product Overview

### 제품 설명
B2B HR SaaS 플랫폼. 한국 중소·중견기업(10~500인)을 대상으로 인사정보 관리, 근태관리, 휴가관리, 급여 계산, 전자계약 등 핵심 HR 기능을 클라우드로 제공한다.

### 핵심 가치
- 엑셀/수기 기반 HR 업무를 자동화하여 HR 담당자의 반복 업무 시간 절감
- 직원 셀프서비스(출퇴근, 휴가 신청, 급여 명세서)로 조직 효율화
- 법적 컴플라이언스 지원 (52시간제, 연차 자동 계산, 급여 명세서 의무화)

### 타겟 고객
| 세그먼트 | 규모 | 특징 | 우선순위 |
|---------|------|------|---------|
| 스타트업 | 1~10인 | 비용 민감, 빠른 도입 원함, 성장 시 전환 기대 | 2 (무료 티어) |
| 중소기업 | 11~100인 | HR 전담자 생기는 시점, 체계화 니즈 | 1 (핵심 타겟) |
| 중견기업 | 100~500인 | 다수 부서/사업장, 고급 기능 필요, 높은 ARPU | 1 (수익 핵심) |
| 대기업/엔터프라이즈 | 500인+ | 맞춤 계약, 긴 세일즈 사이클 | 3 (향후) |

### 현재 상태
- **투자**: 시드 3억원
- **매출**: Pre-revenue
- **팀**: 5~8인 (개발 중심)
- **제약**: 12개월 런웨이 유지 필수

## 3. Revenue Model

### 선택 모델: Freemium + Per-Seat Subscription 하이브리드

| 근거 | 설명 |
|------|------|
| 시드 적합성 | 무료 티어로 CAC 절감, PMF 검증 가속화 |
| 경쟁 차별화 | Flex 무료 없음 → 진입 장벽 낮춤으로 고객 흡수 |
| 자연 확장 | 고객 직원 증가 = 매출 증가 (Net Revenue Expansion) |
| 글로벌 검증 | BambooHR, Gusto 등에서 입증된 모델 |

### 가치 지표(Value Metric)

- **가치 지표**: 관리되는 활성 직원 수 (Active Employee Count)
- **과금 단위**: 월간 활성 직원 수 x 인당 단가
- **정렬 논리**: 직원이 많을수록 HR 관리 복잡도 증가 → 솔루션 가치 증가 → 더 지불하는 것이 합리적

### 기각 대안

| 모델 | 기각 이유 |
|------|----------|
| 순수 구독 (무료 없음) | 브랜드 인지도 없는 시드 단계에서 유료만으로 고객 획득 비현실적 |
| 종량제 (Usage-based) | HR은 사용량 변동 적어 고객 납득 어려움, 매출 예측 곤란 |
| 거래 수수료 | HR에 거래 개념 약함, 급여 이체 수수료는 은행 규제 복잡 |
| 순수 라이선스 | 엔터프라이즈 세일즈 필요 → 시드 팀으로 세일즈 사이클 감당 불가 |

### 경쟁사 대비 포지션

```
가격 (인/월)
8,000원 ──── Flex Pro
6,000원 ──── 원티드 Pro
5,000원 ──── [자사] Pro          ← 경쟁사 중단~하단 침투 전략
4,000원 ──── Flex Starter
3,500원 ──── 페이히어
3,000원 ──── 원티드 기본
2,500원 ──── [자사] Starter     ← Flex 대비 37.5% 저렴
```

**전략**: 가격으로 문을 열고, 제품 경험으로 잠근다. 가격만으로 이기는 것이 아니라 "이 가격에 이 수준의 제품"이라는 가성비 인식을 심는다.

## 6. Risk Assessment

### 상위 3 리스크

#### 리스크 #1: 런웨이 소진 (치명적, 발생 확률: 중~높)

**시나리오**: 보수적 시나리오에서 12개월 말 현금 870만원. 기본 시나리오에서도 3,583만원으로 추가 투자 없이 18개월 이상 생존 어려움.

**대응 방안**:
1. **월 8 시점에서 시리즈 A / 브릿지 투자 유치 시작** (기본 시나리오 현금 ~1.15억 시점)
2. **연간 선결제 적극 유도** (20% 할인) → 현금흐름 선행 확보
3. **Plan B: Burn Rate 2,000만원 "생존 모드"** 사전 준비 (비핵심 지출 삭감 플랜)
4. **KPI 트리거**: 월 6 시점 유료 고객 15사 미달 시 생존 모드 발동

#### 리스크 #2: PMF 미달 + 초기 고Churn (치명적, 발생 확률: 중)

**시나리오**: 제품 완성도 부족 → 고객 이탈 → LTV 하락 → 유닛 이코노믹스 붕괴. HR SaaS는 기능 범위가 넓어 MVP도 상당한 완성도 필요.

**대응 방안**:
1. **MVP 최소화**: 근태 + 휴가 관리만으로 출시 (급여는 Phase 2)
2. **무료 10사 집중 피드백**: 주 2회 배포, 이탈 고객 100% 인터뷰
3. **PMF 지표 정의**: "주 3회 이상 로그인 고객 비율 40%+"를 PMF 달성 기준으로 설정
4. **유료 전환 지연 옵션**: 제품 미성숙 시 무료 베타 연장하고 유료 전환 시기 연기

#### 리스크 #3: 경쟁사 가격 대응 (관리 가능, 발생 확률: 중~낮)

**시나리오**: Flex가 무료 티어 도입하거나 원티드스페이스가 가격 인하하여 가격 우위 무력화.

**대응 방안**:
1. **특정 세그먼트 특화**: "XX산업 전용" 또는 "스타트업 전용"으로 포지셔닝
2. **전환 비용 구축**: 데이터 축적, 워크플로우 커스터마이징, API 연동으로 Lock-in
3. **가격 외 차별화**: UX 우위, 온보딩 속도, HR 담당자 커뮤니티 구축
4. **모니터링**: 경쟁사 가격 변동 월간 추적
