# Phase 3: 수익 모델 설계

## 1. 선택한 수익 모델: 프리미엄(Freemium) + 직원 수 기반 구독(Subscription) 하이브리드

### 선택 이유

| 근거 | 설명 |
|------|------|
| 시드 단계 적합성 | 무료 티어로 초기 고객 획득 비용(CAC) 절감, PMF 검증 가속화 |
| 경쟁 포지셔닝 | Flex는 무료 티어 없음 → 무료 진입 장벽으로 차별화 |
| 직원 수 연동 | 고객의 성장이 곧 매출 성장 (Natural Expansion Revenue) |
| 업그레이드 경로 | 무료→스타터→프로 단계적 전환으로 부담 최소화 |
| 시장 검증 | 글로벌 HR SaaS(BambooHR, Gusto)에서 입증된 모델 |

## 2. 기각한 대안 모델

| 모델 | 기각 이유 |
|------|----------|
| 순수 구독 (무료 티어 없음) | 시드 단계에서 브랜드 인지도 없이 유료 전환 어려움, Flex 대비 후발 열위 |
| 종량제 (Usage-based) | HR SaaS는 사용량 변동이 적어 고객 납득 어려움, 매출 예측도 곤란 |
| 거래 수수료 | HR에 거래 개념이 약함 (급여 이체 수수료 모델은 은행 규제 복잡) |
| 순수 라이선스 | 엔터프라이즈 세일즈 필요 → 시드 팀 규모와 세일즈 사이클 비현실적 |

## 3. 가치 지표 & 과금 단위

| 항목 | 정의 |
|------|------|
| **가치 지표** | 관리되는 직원 수 (Active Employee) |
| **과금 단위** | 월간 활성 직원 수 × 인당 단가 |
| **정렬 논리** | 직원이 많을수록 HR 관리 부담 증가 → 솔루션 가치 증가 → 더 많이 지불하는 것이 합리적 |

| **가격** | 0원 | 2,500원/인/월 | 5,000원/인/월 |
| **대상** | 1~10인 스타트업 | 11~100인 중소기업 | 50~500인 중견기업 |
| **직원 한도** | 10인 | 무제한 | 무제한 |
| **인사정보 관리** | O | O | O |
| **근태관리** | O (기본) | O (고급 규칙) | O (멀티 사업장) |
| **휴가관리** | O (기본) | O (커스텀 정책) | O (고급 정책) |
| **급여 계산** | X | O (기본) | O (세무 연동) |
| **전자 계약** | X | O (월 10건) | O (무제한) |
| **조직도/권한** | 단일 관리자 | 다중 관리자 | 부서별 권한 |
| **리포트/분석** | X | 기본 리포트 | 고급 분석 + 대시보드 |
| **API 연동** | X | X | O |
| **전담 지원** | 이메일 | 채팅 + 이메일 | 전담 매니저 |
| **데이터 내보내기** | X | CSV | CSV + API |
| **연간 결제 할인** | - | 20% (2,000원/인/월) | 20% (4,000원/인/월) |

### 엔터프라이즈 (500인+)
- 별도 견적, 맞춤 계약
- 온프레미스 옵션, SSO, 감사 로그, SLA
- 목표: 연 계약 1,000만원+

## 5. 초기 가격 범위 결정 논리

| 기준 | 분석 |
|------|------|
| **경쟁사 하단** | 시프티 2,000원/인/월 (근태 특화) |
| **경쟁사 중단** | 원티드스페이스 3,000~6,000원/인/월 |
| **경쟁사 상단** | Flex 4,000~8,000원/인/월 (올인원) |
| **자사 포지션** | 경쟁사 하단~중단 (침투 가격 전략) |
| **Starter 2,500원** | Flex 스타터(4,000원) 대비 37.5% 저렴, 시프티(2,000원) 대비 기능 우위 |
| **Pro 5,000원** | Flex 프로(8,000원) 대비 37.5% 저렴, 기능은 80% 수준 |
