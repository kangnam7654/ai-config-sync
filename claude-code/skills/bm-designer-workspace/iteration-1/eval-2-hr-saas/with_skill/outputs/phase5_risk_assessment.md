# Phase 5: 전략 검증 (CSO 리스크 분석)

## 검증 결과 요약

### 1. 수익 모델의 지속 가능성: 조건부 양호

**양호한 점:**
- Freemium + Per-Seat 구독은 HR SaaS에서 검증된 모델 (BambooHR, Gusto 등 글로벌 선례)
- 직원 수 연동 과금은 고객 성장에 따른 자연 매출 확장(Net Revenue Expansion) 가능
- 한국 HR SaaS 시장의 연 15~20% 성장은 후발 주자에도 기회 제공

**우려 사항:**
- Flex의 선점 효과가 강력 (브랜드, 데이터 축적, 채널). 후발 진입 시 단순 가격 경쟁만으로는 부족
- 경쟁사가 무료 티어 도입 시 차별점 상실 위험
- SMB 대상 HR SaaS는 경기 침체 시 가장 먼저 삭감되는 비용 중 하나

### 2. 가격 전략의 적정성: 양호 (단, 하방 리스크 존재)

**양호한 점:**
- Starter 2,500원/인/월은 Flex(4,000원) 대비 37.5% 저렴 → 가격 민감 고객 흡수 가능
- 무료 티어(10인)는 스타트업의 초기 니즈를 충족하면서 제품 경험 제공
- 연간 결제 20% 할인은 현금흐름 안정화와 Churn 감소 효과

**우려 사항:**
- 저가 전략은 "싸니까 쓴다" 고객을 유인 → 이들의 Churn이 높고 LTV가 낮을 수 있음
- Starter→Pro 업그레이드 동기가 충분한지 검증 필요 (기능 차이가 가격 2배를 정당화하는가?)
- 엔터프라이즈 별도 견적은 시드 단계에서 세일즈 팀 부재로 실행 어려움

### 3. 유닛 이코노믹스의 현실성: 주의 필요

**현실적인 부분:**
- LTV/CAC 4.5:1은 건강하지만, 이는 평균 이용 기간 36개월 가정에 기반
- Gross Margin 78%는 SaaS 표준 범위 내

**낙관적 가정 경고:**
- **CAC 100만원 가정**: 시드 단계에서 브랜드 인지도 없이 B2B HR 고객을 월 4사 획득하기 어려울 수 있음. 초기 6개월은 CAC 150~200만원 예상
- **월 Churn 3% 가정**: 한국 SMB SaaS의 현실적 Churn은 3~5%. 제품 완성도가 낮은 초기에는 5%+ 가능
- **ARPU 12.5만원 가정**: 초기 고객은 소규모(20~30인)일 가능성 높아 ARPU 7~8만원 시작 가능성
- **이 세 가정이 동시에 비관적으로 작용하면 보수적 시나리오보다 악화 가능**

### 4. 스케일링 시 비용 구조 변화: 규모의 경제 가능

**긍정적:**
- SaaS 특성상 서버 비용은 고객 수에 비례하지만 한계비용은 체감
- 50사 → 500사 성장 시 인당 지원 비용 감소 (셀프 서비스 비율 증가)
- 개발비는 고정비 → 고객 수 증가 시 인당 개발비 희석

**부정적:**
- 100사+ 시점에서 CS 인력 추가 필요 (고정비 점프)
- 급여 계산 모듈은 법률/세무 변경에 따른 유지보수 비용 지속 발생
- 엔터프라이즈 진입 시 보안 인증(ISMS), 컴플라이언스 비용 발생

### 5. 최대 리스크 3가지

---

#### 리스크 #1: 런웨이 소진 (치명적)

**설명**: 보수적 시나리오에서 12개월 말 현금 870만원. 기본 시나리오에서도 3,583만원으로 추가 투자 없이 18개월 이상 생존 어려움.

**발생 확률**: 중~높 (시드 스타트업의 50%+가 18개월 내 추가 투자 필요)

**대응 방안:**
1. 월 8 시점(기본 시나리오 현금 1.15억)에서 시리즈 A 또는 브릿지 투자 유치 시작
2. 연간 선결제 고객 확보로 현금흐름 선행 확보 (20% 할인은 이 목적에도 부합)
3. Burn Rate를 2,000만원으로 조이는 "생존 모드" 플랜 B 준비
4. KPI 모니터링: 월별 현금 잔액, MRR 성장률, 유료 전환율

---

#### 리스크 #2: PMF 미달 + 높은 초기 Churn (치명적)

**설명**: 제품 완성도 부족 → 고객 이탈 → LTV 하락 → 유닛 이코노믹스 붕괴. 초기 6개월은 제품이 불완전할 확률 높음.

**발생 확률**: 중 (HR SaaS는 기능 범위가 넓어 MVP도 상당한 완성도 필요)

**대응 방안:**
1. MVP 기능 범위를 좁혀 "근태 + 휴가 관리"만으로 시작 (급여는 후순위)
2. 무료 10사에서 집중적 피드백 수집 → 주 2회 배포 사이클
3. 초기 3개월은 Churn을 KPI가 아닌 "학습 지표"로 관리 (이탈 이유 100% 인터뷰)
4. 제품 준비도가 부족하면 무료 베타로 전환하여 유료 전환 시기를 늦추는 것도 옵션

---

#### 리스크 #3: 경쟁사의 가격 대응 (관리 가능)

**설명**: Flex나 원티드스페이스가 가격 인하 또는 무료 티어 도입으로 가격 우위 무력화.

**발생 확률**: 중~낮 (Flex는 이미 가격 인상 추세, 가격 인하 가능성 낮음. 단, 무료 티어 도입 가능성은 있음)

**대응 방안:**
1. 가격만이 아닌 "특정 세그먼트 특화"로 차별화 (예: 스타트업 전용, 특정 산업 전용)
2. 전환 비용(Switching Cost) 구축: 데이터 축적, 워크플로우 커스터마이징, API 연동
3. 커뮤니티/콘텐츠로 브랜드 락인 (HR 담당자 커뮤니티, 노동법 가이드 등)
4. 가격 이외의 가치 제안 강화 (UX, 모바일, 온보딩 속도)
