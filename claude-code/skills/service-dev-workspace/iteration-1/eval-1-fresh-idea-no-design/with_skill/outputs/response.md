## Phase 0 실행: 아이디어 구조화 (Path B - 사용자 직접 입력)

사용자가 제공한 아이디어를 4가지 필수 항목으로 정리합니다:

### 서비스 개요

개발자가 자주 사용하는 코드 스니펫을 로컬에 저장하고, 태그/키워드로 빠르게 검색하며, 팀원과 공유할 수 있는 CLI 도구.

### 핵심 기능

1. **스니펫 저장**: 코드 조각을 제목, 언어, 태그와 함께 저장 (`snippet save`)
2. **스니펫 검색**: 키워드, 태그, 언어 필터로 스니펫을 검색 (`snippet search`, `snippet find`)
3. **스니펫 조회/복사**: 저장된 스니펫을 터미널에 출력하거나 클립보드에 복사 (`snippet get`, `snippet copy`)
4. **스니펫 편집/삭제**: 기존 스니펫 수정 및 삭제 (`snippet edit`, `snippet rm`)
5. **팀 공유**: Git 기반 원격 저장소를 통한 팀 스니펫 동기화 (`snippet sync`, `snippet push`, `snippet pull`)
6. **스니펫 목록**: 저장된 스니펫을 태그별/언어별로 그룹화하여 목록 출력 (`snippet list`)
7. **임포트/익스포트**: 스니펫을 JSON/YAML로 내보내기/가져오기 (`snippet export`, `snippet import`)

### 타겟 사용자

- **1차 타겟**: 매일 반복적인 코드 패턴(boilerplate, 설정 파일 조각, 유틸 함수 등)을 사용하는 개발자 (주니어~시니어, 모든 직군)
- **규모**: 5~30명 규모의 개발팀
- **고충**: 자주 쓰는 코드를 매번 검색하거나, Slack/노션에 흩어진 스니펫을 찾느라 시간을 허비함. 기존 스니펫 매니저(Gist, SnippetsLab 등)는 GUI 기반이라 터미널 워크플로우와 단절됨.

### BM 요약

- **무료 (오픈소스)**: 로컬 스니펫 저장/검색, Git 기반 팀 동기화 — 핵심 기능 전체를 무료로 제공
- 이 도구는 **개발자 생산성 도구**로, 오픈소스 CLI 프로젝트로 운영. 수익화보다는 커뮤니티 확산과 개인 포트폴리오/팀 내부 도구로 가치를 둠.
- (선택적 확장) 향후 유료 SaaS 확장 시: 클라우드 호스팅 팀 허브, 조직 관리, 접근 제어 — 월 $5/user

## Phase 0 게이트: 승인 요청

위 4가지 산출물(서비스 개요, 핵심 기능, 타겟 사용자, BM 요약)을 확인해 주세요.

추가로 확인이 필요한 사항:

1. **서비스 이름**: `snippet-hub`, `code-vault`, 또는 다른 이름을 원하시나요? (산출물 저장 경로 `docs/{service-name}/`에 사용됩니다)
2. **핵심 기능 범위**: 위 7개 기능이 적절한가요? 추가하거나 제거할 기능이 있나요?
3. **팀 공유 방식**: Git 기반 동기화를 제안했는데, 다른 방식(예: 자체 서버, S3 등)을 선호하시나요?

승인해 주시면 Phase 1(Plan)으로 진행하겠습니다. Phase 2(Design)는 CLI 도구이므로 건너뜁니다.
