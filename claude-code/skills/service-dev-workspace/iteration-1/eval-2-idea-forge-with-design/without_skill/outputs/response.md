# 반려동물 건강 관리 SaaS - 서비스 개발 시작

좋습니다! Idea Forge에서 검증 완료된 '반려동물 건강 관리 SaaS' 아이디어로 서비스 개발을 시작하겠습니다.

## 현재 상태 확인

- Idea Forge 검증: 완료 (CEO -> Researcher -> CSO 삼각 검증 통과)
- BM 설계: 완료 (bm-designer 파이프라인 거쳐 BM Score 정량 평가 완료)
- 플랫폼: 모바일 웹
- 디자인: 필요

## Design-First Development 원칙에 따른 진행

구현 코드를 작성하기 전에 설계문서를 먼저 작성해야 합니다. 아래 순서로 진행하겠습니다.

### 1단계: 설계문서 작성 (구현 전 필수)

`docs/pet-health-saas/` 경로에 설계문서를 작성합니다. 포함 내용:

| 섹션 | 내용 |
|------|------|
| 목적 | 반려동물 건강 관리 SaaS가 해결하는 문제 + 성공 기준 |
| 아키텍처 | 시스템 컴포넌트 다이어그램 (Mermaid) - 프론트엔드, 백엔드, DB, 외부 서비스 |
| 데이터 흐름 | 사용자 입력 -> 처리 -> 출력 경로 단계별 기술 |
| API 설계 | 주요 엔드포인트 + 요청/응답 스키마 |
| 파일 구조 | 프로젝트 디렉토리 트리 |
| 의사결정 근거 | 기술 스택 선택 이유 + 기각한 대안 |

### 2단계: 디자인 작업

모바일 웹이므로 UI/UX 디자인이 필요합니다. 설계문서에 아래 내용을 포함합니다:

- 주요 화면 목록 및 와이어프레임
- 디자인 시스템 (색상, 타이포그래피, 컴포넌트)
- 사용자 플로우 다이어그램

### 3단계: 사용자 승인 후 구현

설계문서가 완성되면 리뷰를 요청드리고, 승인 후 구현을 시작합니다.

## 진행을 위해 확인이 필요한 사항

설계문서 작성을 시작하기 위해 몇 가지 확인이 필요합니다:

### 1. BM 검증 결과 공유
Idea Forge에서 나온 BM 설계 결과를 공유해주시면 설계문서에 반영하겠습니다. 특히:
- 타겟 고객 세그먼트
- 핵심 가치 제안
- 수익 모델 (구독제, 프리미엄 등)
- BM Score 결과

### 2. 기술 스택 선호도
모바일 웹 기준으로 아래 선택지가 있습니다:
- **프론트엔드**: Next.js / Nuxt / SvelteKit 등
- **백엔드**: FastAPI(Python) / Express(Node) / Go 등
- **DB**: PostgreSQL / MongoDB / Supabase 등
- 선호하시는 스택이 있으신가요?

### 3. 디자인 도구 및 방식
- Figma 파일이 연결되어 있으므로 Figma에서 디자인을 진행할 수 있습니다
- 기존 디자인 시스템이나 참고할 디자인이 있나요?
- 디자인 우선순위: 전체 화면을 먼저 설계할지, MVP 핵심 화면만 먼저 할지?

### 4. MVP 범위 정의
첫 번째 릴리스에 포함할 핵심 기능을 정해야 합니다. 예상되는 후보:
- 반려동물 프로필 등록/관리
- 건강 기록 (체중, 예방접종, 병원 방문)
- 알림/리마인더 (예방접종 일정, 약 복용)
- 건강 리포트/대시보드

어떤 기능을 MVP에 포함할지 결정해주세요.

### 5. 프로젝트 저장소
새 저장소를 만들어야 합니다:
- 저장소 이름 (예: `pet-health-saas`)
- GitHub organization 또는 개인 계정?
- 모노레포 vs 프론트/백 분리?
