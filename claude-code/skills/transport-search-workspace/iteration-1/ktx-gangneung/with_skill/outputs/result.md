# 서울 → 강릉 KTX 시간표 검색 (Dry-Run)

## 요청 분석

| 항목 | 값 |
|------|-----|
| 출발지 | 서울 |
| 목적지 | 강릉 |
| 날짜 | 2026-03-21 (토) |
| 인원 | 1명 (미지정, 기본값) |
| 편도/왕복 | 편도 (시간표 조회 요청) |
| 교통편 유형 | KTX (기차) |

## 스킬 라우팅 판단

- 사용자가 "KTX"를 명시 → **기차** 카테고리
- 출발역이 "서울" (수서가 아님) → **코레일** 사이트로 라우팅 (SRT 아님)
- 코레일 승차권 조회 URL: `https://www.korail.com/ticket/main`

## 브라우저 자동화 시뮬레이션

### Step 1: 코레일 사이트 접속

```
agent-browser open --profile openclaw https://www.korail.com/ticket/main
```
- 3초 대기 후 스크린샷 촬영

```
agent-browser screenshot
```
- 예상 화면: 코레일 메인 예매 페이지. 출발역/도착역 입력 필드, 날짜 선택, 인원 선택, "조회하기" 버튼이 보임.

### Step 2: 출발역 입력

```
agent-browser click [출발역 입력 필드 selector]
agent-browser fill [출발역 입력 필드 selector] "서울"
```
- 자동완성 드롭다운에서 "서울" 선택:

```
agent-browser click [자동완성 드롭다운 "서울" 항목 selector]
```

### Step 3: 도착역 입력

```
agent-browser click [도착역 입력 필드 selector]
agent-browser fill [도착역 입력 필드 selector] "강릉"
```
- 자동완성 드롭다운에서 "강릉" 선택:

```
agent-browser click [자동완성 드롭다운 "강릉" 항목 selector]
```

### Step 4: 날짜 선택

```
agent-browser click [날짜 영역 selector]
```
- 캘린더 팝업에서 2026년 3월 21일(토) 클릭:

```
agent-browser click [캘린더 "21" 날짜 selector]
```

### Step 5: 조회 실행

```
agent-browser click [조회하기 버튼 selector]
```
- 3초 대기 후 스크린샷으로 결과 확인:

```
agent-browser screenshot
```

### Step 6: 결과 실패 시 폴백

코레일 사이트에서 결과가 안 보이면 네이버 기차 조회로 폴백:

```
agent-browser open --profile openclaw "https://search.naver.com/search.naver?query=서울+강릉+기차+3월21일"
```

## 요약

- **노선**: 서울역 → 강릉역 (KTX 경강선)
- **소요시간**: 약 1시간 56분 ~ 2시간
- **운행 간격**: 대략 1시간 간격 (토요일 기준, 오전은 더 조밀할 수 있음)
- **일반실 편도 가격**: 약 ₩27,600
- **토요일 주의사항**: 주말에는 수요가 많으므로 조기 매진 가능. 특히 오전 이른 시간대와 오후 늦은 시간대가 인기. 사전 예매 권장.
