# SRT 수서 → 부산 검색 결과 (Dry-Run)

## 요청 파싱

| 항목 | 값 |
|------|-----|
| 출발역 | 수서 |
| 도착역 | 부산 |
| 날짜 | 2026-03-21 (내일) |
| 인원 | 2명 (어른) |
| 교통편 유형 | SRT (출발역이 수서 + 사용자가 SRT 명시) |
| 편도/왕복 | 편도 (기차는 편도 가격이 기본) |

## 스킬 라우팅 판단

출발역이 **수서**이고 사용자가 **SRT**를 명시했으므로 스킬 규칙에 따라 SRT 승차권 조회 사이트로 라우팅한다.

> "출발역이 수서이거나 사용자가 SRT를 명시하면 → SRT 검색. 그 외 → 코레일 검색."

## 브라우저 자동화 시뮬레이션

### Step 1: SRT 승차권 조회 페이지 열기

```
agent-browser open --profile openclaw https://etk.srail.kr/hpg/hra/01/selectScheduleList.do
```

3초 대기 후 폼 구조 확인:

```
agent-browser screenshot
```

예상 화면: SRT 승차권 예매 페이지. 출발역, 도착역, 출발일, 출발시간, 인원 선택 폼이 표시됨.

### Step 2: 출발역 입력

```
agent-browser click "#dptRsStnCdNm"
agent-browser fill "#dptRsStnCdNm" "수서"
```

자동완성 드롭다운에서 "수서" 선택:

```
agent-browser click ".autocomplete-item:first-child"
```

### Step 3: 도착역 입력

```
agent-browser click "#arvRsStnCdNm"
agent-browser fill "#arvRsStnCdNm" "부산"
```

자동완성 드롭다운에서 "부산" 선택:

```
agent-browser click ".autocomplete-item:first-child"
```

### Step 4: 날짜 선택

캘린더에서 2026년 3월 21일 선택:

```
agent-browser click "#dptDt"
agent-browser click "[data-date='20260321']"
```

### Step 5: 인원 변경

기본값 어른 1명 → 2명으로 변경 (+ 버튼 1회 클릭):

```
agent-browser click ".adult-plus-btn"
```

### Step 6: 조회 실행

```
agent-browser click "#search-btn"
```

3초 대기 후 결과 스크린샷:

```
agent-browser screenshot
```
