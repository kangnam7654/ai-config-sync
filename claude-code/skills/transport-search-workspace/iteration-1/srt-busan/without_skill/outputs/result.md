## 접근 방법 (Dry-Run)

### 1단계: 사이트 선택

SRT 열차 시간표 및 예매는 **SRT 공식 웹사이트**를 통해 조회합니다.

- **URL**: `https://etk.srail.kr/hpg/hra/01/selectScheduleList.do`
- **대안 URL**: `https://etk.srail.kr/` (메인 페이지에서 조회)

### 2단계: 조회 방법

SRT 공식 사이트의 열차 조회 API는 POST 요청으로 동작하며, 아래 파라미터가 필요합니다:

```
POST https://etk.srail.kr/hpg/hra/01/selectScheduleList.do

파라미터:
- dptRsStnCd: 0551 (수서역 코드)
- arvRsStnCd: 0020 (부산역 코드)
- stlbTrnClsfCd: 17 (SRT)
- psgNum: 2 (인원)
- seatAttCd: 015 (일반석)
- is498Tm498: Y
- dptDt: 20260321 (출발일)
- dptTm: 000000 (조회 시작 시간)
- chtnDvCd: 1 (편도)
- arriveTime: N
```

### 3단계: WebFetch로 조회 시도

실제 실행 시에는 `WebFetch`로 위 URL에 접근하거나, SRT 사이트가 동적 렌더링이 필요한 경우 `agent-browser`를 사용해야 합니다.

> **참고**: SRT 사이트는 서버사이드 렌더링 + JavaScript 의존도가 높아 단순 WebFetch로는 조회가 어려울 수 있습니다. 실제로는 `agent-browser`를 사용하여 폼을 채우고 조회 버튼을 클릭하는 방식이 필요할 가능성이 높습니다.
