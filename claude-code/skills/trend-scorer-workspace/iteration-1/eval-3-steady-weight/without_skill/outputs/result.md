## 1. GitHub Copilot

### 데이터 근거

- **사용자 기반**: 2,000만 누적 사용자 (2025.07), 유료 구독자 470만 (2026.01), YoY 75% 성장
- **시장 점유율**: AI 코딩 도구 유료 시장 42% (2026 기준) -- 여전히 1위
- **엔터프라이즈**: Fortune 100 기업의 90%가 배포, 50,000개 이상 조직에서 사용
- **리텐션**: 20주 후 89% 사용자가 계속 사용 (2025년 데이터)
- **성장 패턴**: 분기별 구독자 30% 성장 유지 (2024), 엔터프라이즈 고객 QoQ 75% 성장 (2025 Q2)
- **"Most Loved" 비율**: 9% (2026 초)
- **약점**: 개발자 선호도에서 Cursor, Claude Code에 밀림. 기술적 우위보다 기업 정책에 의한 잔존 비중이 높음

## 분석 요약

### "꾸준한 관심" 관점에서의 핵심 인사이트

1. **Copilot의 역설**: "Most Loved"는 9%로 최하위지만, 꾸준한 관심에서는 1위다. 이는 엔터프라이즈 고착 효과와 MS/GitHub 생태계 통합 덕분이다. 개발자가 열광하지 않아도 이탈하지 않는다. 2021년 출시 이후 4년간 QoQ 30% 성장을 안정적으로 유지한 것은 "꾸준함"의 정의에 가장 부합한다.

2. **Cursor의 균형**: 폭발적 성장(3개월 만에 ARR 2배)은 인상적이지만, "꾸준한 관심"이라는 렌즈에서는 양면적이다. 급성장은 급락의 가능성도 내포한다. 그러나 89%의 높은 리텐션과 엔터프라이즈 매출 비중 60%가 단순 유행이 아님을 시사한다.

3. **Claude Code의 잠재력**: 46% "Most Loved"는 압도적 개발자 사랑을 보여주지만, 출시 8개월로는 "꾸준함"을 증명하기엔 트랙 레코드가 짧다. 81%의 상대적으로 낮은 리텐션과 사용량 제한 불만은 장기 관심 유지에 리스크 요인이다. 다만, 이 궤적이 유지된다면 6개월 후에는 판도가 바뀔 수 있다.

4. **멀티툴 시대**: 평균 2.3개 도구를 동시에 사용하는 현 개발자 행태에서, "꾸준한 관심"은 단독 사용 충성도보다 "포트폴리오 내 잔존 여부"로 해석해야 한다. 이 관점에서 세 도구 모두 각자의 역할로 살아남을 가능성이 높다.
