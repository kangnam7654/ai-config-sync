- **Trending 근거**:

([Google Cloud 2026 AI Agent Trends Report](https://cloud.google.com/resources/content/ai-agent-trends-2026), [오픈소스 AI 에이전트 열풍](https://dev-post.com/open-source-ai-agent-github-trending-2026/))
  - SBI 8 — Reddit r/ArtificialIntelligence에서 AI 에이전트 논의가 활발하며, Claude Code만 226개 커뮤니티 언급이 확인됨. AI 에이전트 전용 소셜 플랫폼 Moltbook이 2026.1.28 런칭되어 화제. 개발자 커뮤니티에서 비용 대비 효과(ROI) 논쟁이 뜨거움. ([Best AI Agents Reddit 2026](https://www.aitooldiscovery.com/guides/best-ai-agents-reddit), [Moltbook](https://www.starkinsider.com/2026/02/ai-agents-moltbook-human-ai-collaboration.html))
  - NFI 9 — 2026년 3월 한 달에만 267개의 새로운 AI 모델 발표. 에이전틱 AI 관련 기사가 환경감시일보, AI 타임즈, MIT Technology Review Korea, 한국경제 등 주요 언론사에 연일 보도. Google, Microsoft, Anthropic 등 빅테크의 에이전트 관련 발표가 지속. ([AI 에이전트 혁명](https://www.dait90000.com/2026/03/AIagentrevolution.html), [SK AX 에이전트 AI 트렌드](https://www.skax.co.kr/insight/trend/3624))

### 2위: MCP - Model Context Protocol (Final: 6.22)

- **왜 높은가**: 2024년 11월 Anthropic이 출시한 이후 불과 16개월 만에 OpenAI, Google, Microsoft가 모두 채택한 사실상의 표준(de facto standard)으로 자리잡았다. 97M+ 월간 SDK 다운로드와 10,000+ 공개 MCP 서버가 생태계의 급속한 성장을 증명한다.
- **Trending 근거**:
  - SVGR 7 — MCP 서버 다운로드가 2024.11 약 10만에서 2025.4 800만+으로 폭증한 이력이 있으며, 2026.2 기준 공식 레지스트리에 6,400+ 서버 등록. 다만 초기 폭발적 성장 이후 현재는 안정 성장 국면으로 전환 중. Perplexity CTO가 MCP에서 이탈을 선언하는 등 "MCP 피로감" 논의도 등장. ([The New Stack MCP Roadmap 2026](https://thenewstack.io/model-context-protocol-roadmap-2026/), [MCP is Dead; Long Live MCP!](https://chrlschn.dev/blog/2026/03/mcp-is-dead-long-live-mcp/))
  - SBI 7 — Reddit에서 "MCP is worth learning early" 등 개발자 학습 권장 논의가 활발. 기술 커뮤니티에서 MCP의 장단점(컨텍스트 윈도우 40-50% 소비, 인증 마찰)에 대한 깊이 있는 토론이 진행 중. 다만 일반 대중 SNS보다는 개발자 커뮤니티 중심의 논의. ([CIO - Why MCP is on every executive agenda](https://www.cio.com/article/4136548/why-model-context-protocol-is-suddenly-on-every-executive-agenda.html), [Wikipedia MCP](https://en.wikipedia.org/wiki/Model_Context_Protocol))
  - NFI 8 — 2026년 3월 Google Colab MCP 서버 공개(3/19), SurePath AI의 MCP 정책 컨트롤 발표(3/12), 2026 MCP 로드맵 공개 등 주 단위로 주요 뉴스 발생. IBM, Red Hat, Cloudflare 등 대형 기업이 MCP 가이드를 발간. ([Google Colab MCP](https://www.marktechpost.com/2026/03/19/google-colab-now-has-an-open-source-mcp-model-context-protocol-server-use-colab-runtimes-with-gpus-from-any-local-ai-agent/?amp=), [SurePath AI MCP](https://www.prnewswire.com/news-releases/surepath-ai-advances-real-time-model-context-protocol-mcp-policy-controls-to-govern-ai-actions-302709875.html))
- **Steady 근거**:
  - STB 5 — 2024.11 출시 후 급성장기를 거쳐 현재 성장 안정화 단계로 진입 중이나, 아직 16개월밖에 되지 않아 장기 안정성 판단에 한계. Linux Foundation 산하 Agentic AI Foundation으로 이관되어 거버넌스는 안정화되었으나, "MCP is Dead" 담론과 Perplexity CTO의 이탈 선언 등 변동성 요인 존재. ([Pento - A Year of MCP](https://www.pento.ai/blog/a-year-of-mcp-2025-review), [Versalence - Long Live MCP](https://blogs.versalence.ai/mcp-model-context-protocol-evolution-2026))
  - VOL 7 — 97M+ 월간 SDK 다운로드, 10,000+ 공개 MCP 서버, Anthropic/OpenAI/Google/Microsoft 4대 빅테크 공식 지원. 한국에서도 네이버 뉴스 MCP, 네이버 검색광고 MCP 등 한국어 생태계가 형성 중. 다만 "AI agent"보다 전문 개발자 중심으로 검색 볼륨은 상대적으로 낮음. ([CData MCP 2026](https://www.cdata.com/blog/2026-year-enterprise-ready-mcp-adoption), [네이버 검색광고 MCP](https://retn.kr/blog/naver-searchad-mcp-seo-automation/))
  - SEA 2 — 2024.11 출시 이후 16개월 데이터만 존재하여 연간 패턴 판단 불가. 기술 프로토콜 특성상 계절적 반복 패턴이 나타날 가능성은 매우 낮음. 주요 AI 컨퍼런스(I/O, Build, re:Invent)에 맞춰 업데이트가 발표되지만 아직 연간 주기 확인 불가.
- **주의점**: MCP가 컨텍스트 윈도우의 40-50%를 소비한다는 기술적 한계가 보고되고 있으며, Perplexity CTO가 MCP 대신 전통적 API/CLI 도구로의 회귀를 선언하는 등 기술적 회의론이 일부 존재한다. 프로토콜의 장기 생존 여부는 이러한 성능 문제 해결에 달려 있다.

## 비교 분석 요약

| 비교 항목 | AI Agent | MCP |
|-----------|----------|-----|
| Final Score | **7.44** | 6.22 |
| Trending Score (TS) | **8.25** | 7.25 |
| Steady Score (SS) | **6.45** | 4.95 |
| 핵심 강점 | 시장 규모/성장률, 뉴스 노출도, 범용적 관심 | 기술 표준으로의 빠른 정착, 빅테크 4사 동시 채택 |
| 핵심 약점 | 버블론/ROI 미검증, 마케팅 남용 우려 | 역사가 짧아 안정성 미검증, 기술적 한계(토큰 소비) |
| 관심 범위 | 일반 대중 + 기업 + 개발자 | 개발자 + 기업 (기술 전문가 중심) |
| 시장 단계 | 폭발적 성장기 -> 실전 배치 확대기 | 초기 급성장 후 안정 성장 전환기 |

**결론**: AI Agent가 TS, SS 모두에서 MCP를 앞서며 종합 1위를 차지했다. AI Agent는 기술 주제를 넘어 비즈니스/소비자 영역까지 확산된 범용 트렌드인 반면, MCP는 AI Agent 생태계를 지탱하는 인프라 프로토콜로서의 성격이 강하다. 실제로 MCP는 AI Agent의 하위 레이어(연결 표준)이므로 AI Agent 트렌드가 지속되는 한 MCP도 함께 성장할 가능성이 높다. 다만 MCP는 기술적 한계(컨텍스트 윈도우 소비, 인증 문제)와 대안 등장 가능성이 변수로 남아 있다.
