# Slide (PPTX) Design Review Criteria

ui-review 스킬이 프레젠테이션 슬라이드를 평가할 때 참조하는 기준.
슬라이드는 앱/웹과 달리 인터랙션이 없으므로, 기존 8차원 중 일부를 슬라이드 맥락에 맞게 재해석한다.

---

## 1. 레이아웃 원칙

### Rule of Thirds
- 슬라이드를 3x3 그리드로 나눠 핵심 요소를 교차점에 배치
- 중앙 정렬만 반복하면 단조로움 → 의도적 비대칭으로 시선 유도

### Z-Pattern 읽기 흐름
- 좌상단 → 우상단 → 좌하단 → 우하단 순서로 시선 이동
- 가장 중요한 메시지는 좌상단 또는 중앙에 배치
- CTA(Call-to-Action)는 우하단에 배치

### 단일 메시지 원칙
- 한 슬라이드에 하나의 핵심 메시지만 전달
- "이 슬라이드가 말하고자 하는 것을 한 문장으로 요약할 수 있는가?"
- 메시지가 2개 이상이면 슬라이드를 분리

### 텍스트 분량 제한
- 본문 텍스트: 최대 6–7줄 (bullet point 포함)
- 한 줄 길이: 최대 40–50자 (한글 기준)
- 한 슬라이드 총 단어 수: 최대 40–50단어 (영문 기준), 30–40자 (한글 기준)
- "5초 규칙": 청중이 5초 내에 슬라이드 내용을 파악할 수 있어야 함

---

## 2. 타이포그래피

### 폰트 크기 가이드

| 용도 | 최소 크기 | 권장 크기 | 최대 크기 |
|------|-----------|-----------|-----------|
| 슬라이드 제목 | 28pt | 32–40pt | 44pt |
| 부제목 | 20pt | 24–28pt | 32pt |
| 본문 텍스트 | 18pt | 20–24pt | 28pt |
| 캡션/주석 | 12pt | 14–16pt | 18pt |
| 데이터 레이블 | 10pt | 12–14pt | 16pt |

### 폰트 선택 규칙
- **산세리프 우선**: Pretendard, Noto Sans KR, Apple SD Gothic Neo, Arial, Helvetica
- 세리프는 격식 있는 프레젠테이션에서만 (연간 보고서, 학술 발표)
- **최대 2개 폰트**: 제목용 + 본문용 (같은 패밀리의 weight 변화가 이상적)
- 장식 폰트(Script, Display)는 커버 슬라이드에서만, 본문에 절대 사용 금지

### 텍스트 가독성
- 행간(line spacing): 1.2–1.5배 (PowerPoint 기본 1.0은 너무 좁음)
- 자간(letter spacing): 0–50 범위 (너무 넓으면 산만)
- 좌측 정렬 우선 (양쪽 정렬은 단어 간격 불균일 유발)
- 모든 대문자 텍스트: 짧은 제목/레이블에만 허용

---

## 3. 비주얼 밸런스

### 이미지:텍스트 비율
- 권장: 60:40 (이미지 > 텍스트)
- 텍스트만 있는 슬라이드: 최소 하나의 비주얼 요소 추가 (아이콘, 다이어그램, 숫자 강조)
- 이미지만 있는 슬라이드: 허용 (풀블리드 이미지 + 오버레이 텍스트)

### 여백 (Margin & Padding)
- 슬라이드 가장자리: 최소 0.5인치 (1.27cm) 여백
- 요소 간 간격: 최소 0.25인치 (0.64cm)
- 그룹 내 요소 간격 < 그룹 간 간격 (근접성 원칙)
- 빈 공간은 "비어 있는 것"이 아니라 "의도적 여백" — 채우려 하지 마라

### 시각적 무게 균형
- 큰 이미지가 한쪽에 있으면 반대편에 텍스트 블록으로 균형
- 색이 진한 요소는 시각적 무게가 크므로 분산 배치
- 슬라이드 하단이 무거우면 불안정한 느낌 → 상단에 앵커 요소 배치

---

## 4. 컬러 팔레트

### 색상 수 제한
- **3–5색**: 주요 색상 1개 + 보조 색상 1–2개 + 중립 색상 1–2개
- 배경: 흰색/밝은 회색 (라이트 테마) 또는 진한 회색/남색 (다크 테마)
- 텍스트: #333333 (라이트 배경) 또는 #EEEEEE (다크 배경) — 순수 흑/백 피하기

### 배경-전경 대비
- 텍스트-배경 대비: 최소 4.5:1 (WCAG AA 기준, 프로젝터 환경에서는 더 높은 대비 권장)
- 이미지 위 텍스트: 반투명 오버레이 필수 (`rgba(0,0,0,0.4–0.6)`)
- 그래디언트 배경 위 텍스트: 가장 밝은 영역에서도 대비 확인

### 브랜드 컬러 활용
- 브랜드 컬러가 있으면 Primary로 사용
- 브랜드 컬러가 채도 높은 경우: 포인트에만 사용, 배경에는 tint/shade 변형
- 경쟁사/파트너 언급 시: 해당 브랜드 컬러 차용 가능 (맥락 명확 시)

### 색상 의미론
- 빨강: 위험, 감소, 부정적 → 긍정적 데이터에 사용 금지
- 초록: 성장, 증가, 긍정적
- 파랑: 신뢰, 안정, 기업
- 주황/노랑: 주의, 경고, 강조
- 회색: 중립, 비활성, 비교 기준

## 5. 데이터 시각화

### 차트 유형별 적합 사용처

| 차트 유형 | 적합한 데이터 | 부적합한 경우 |
|-----------|--------------|---------------|
| 막대 차트 (Bar) | 카테고리별 비교, 순위 | 시간 추이 (라인 사용) |
| 라인 차트 (Line) | 시간에 따른 변화, 추세 | 카테고리 비교 (막대 사용) |
| 파이 차트 (Pie) | 전체 대비 비율 (최대 5조각) | 6개 이상 카테고리, 작은 차이 비교 |
| 도넛 차트 (Donut) | 단일 KPI 강조 + 비율 | 다중 비교 |
| 영역 차트 (Area) | 누적 변화, 구성 비율 추이 | 겹치는 데이터 (가려짐) |
| 산점도 (Scatter) | 상관관계, 분포 | 시간 추이, 카테고리 비교 |
| 워터폴 (Waterfall) | 증감 분해, 요인 분석 | 단순 비교 |

### 범례 위치
- 차트 내부 직접 레이블 우선 (범례보다 직관적)
- 범례 필요 시: 차트 상단 또는 우측 (하단은 잘림 위험)
- 범례 항목 최대 5개 (초과 시 "기타"로 그룹화)

### 장식 제거 (Data-Ink Ratio)
- 3D 효과 금지 (왜곡 유발)
- 그리드 라인: 최소화 또는 제거 (밝은 회색으로 약하게)
- 축 레이블: 필요한 것만, 단위 명시
- 데이터 레이블: 핵심 포인트에만 (모든 막대에 숫자 X)
- 불필요한 배경 색상/패턴 제거

### 색상 사용
- 강조할 데이터 포인트만 브랜드/강조 색상, 나머지는 회색
- 색맹 대응: 색상만으로 구분하지 말고 패턴/레이블 병용
- 순차적 데이터: 같은 색상 계열의 밝기 변화 (sequential palette)
- 대비 데이터: 보색 관계 팔레트 (diverging palette)

## 6. 기존 8차원 → 슬라이드 맥락 매핑

슬라이드는 앱/웹과 달리 **정적**이고 **프레젠테이션 맥락**에서 소비된다.
아래 표에 따라 기존 8차원을 슬라이드에 맞게 재해석하여 평가한다.

| # | 원래 차원 | 슬라이드 재해석 | 핵심 평가 포인트 |
|---|-----------|----------------|-----------------|
| 1 | Visual Hierarchy | **메시지 계층** | 핵심 메시지가 가장 눈에 띄는가? 제목→본문→보조 텍스트 크기 차이 명확? |
| 2 | Color & Contrast | **가독성 & 팔레트** | 프로젝터/화면에서 읽히는 대비? 색상 3–5개 제한? 의미론적 색상 사용? |
| 3 | Typography | **타이포그래피** | 폰트 2개 이내? 크기 가이드 준수? 행간 적절? |
| 4 | Spacing & Alignment | **레이아웃 정돈** | 여백 0.5인치 이상? 요소 정렬 일관? 근접성 원칙? |
| 5 | Layout & Composition | **비주얼 밸런스** | 이미지:텍스트 비율? 시각적 무게 균형? 빈 공간 활용? |
| 6 | Navigation & Affordance | **슬라이드 흐름** | 슬라이드 간 논리적 순서? 섹션 구분? 페이지 번호? 목차 슬라이드? |
| 7 | Consistency | **디자인 일관성** | 슬라이드 간 레이아웃/색상/폰트 통일? 마스터 슬라이드 활용? |
| 8 | Trend Alignment | **프레젠테이션 트렌드** | 미니멀 디자인? 대형 비주얼? 데이터 스토리텔링? 인포그래픽 활용? |

### 차원별 가중치 조정 (슬라이드)

| 차원 | 앱/웹 기본 | 슬라이드 조정 | 이유 |
|------|-----------|-------------|------|
| Visual Hierarchy (메시지 계층) | 1.0 | 1.5 | 한 슬라이드 = 한 메시지. 계층 실패 = 슬라이드 실패 |
| Color & Contrast (가독성) | 1.0 | 1.3 | 프로젝터 환경에서 대비 특히 중요 |
| Typography | 1.0 | 1.3 | 먼 거리에서 읽히는 크기 필수 |
| Spacing & Alignment | 1.0 | 1.2 | 정돈되지 않은 슬라이드는 신뢰 감소 |
| Layout & Composition | 1.0 | 1.2 | 비주얼 밸런스가 전문성 인상 좌우 |
| Navigation & Affordance (흐름) | 1.0 | 0.7 | 인터랙션 없으므로 가중치 감소 |
| Consistency | 1.0 | 1.3 | 슬라이드 간 불일치가 매우 눈에 띔 |
| Trend Alignment | 1.0 | 0.8 | 트렌드보다 명확한 전달이 우선 |

### 슬라이드 전용 추가 체크포인트

- [ ] 커버 슬라이드에 제목, 발표자, 날짜, 로고 포함
- [ ] 목차 또는 아젠다 슬라이드 존재 (10장 이상일 때)
- [ ] 섹션 구분 슬라이드 존재 (15장 이상일 때)
- [ ] 마지막 슬라이드: Q&A, 요약, 또는 CTA
- [ ] 슬라이드 번호 표시 (총 수 포함: "3/15")
- [ ] 애니메이션/전환: 있다면 통일, 과도한 효과 없음
- [ ] 발표자 노트: 키 포인트 요약 (리뷰 범위 밖이나 존재 여부 확인)
