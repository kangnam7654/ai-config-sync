# Trend Scorer

인터넷 주제의 **인기도(Trending)**와 **지속성(Steady)**을 정량적으로 평가해 종합 점수로 랭킹한다.

핵심 가치: "감"이 아니라 "수치"로 판단한다. 동일한 주제를 시간을 두고 재평가하면 변화 추이까지 추적할 수 있다.

---

## 범위

### IN SCOPE
- 인터넷 주제의 Trending/Steady 점수를 6개 지표로 산출하고 랭킹 테이블을 출력한다
- 사용자 지정 주제 또는 자동 발굴 주제를 평가한다
- WebSearch/WebFetch로 공개 데이터를 수집한다
- /loop 연계 시 날짜별 리포트를 파일로 저장한다
- 이전 리포트가 존재하면 순위 변동을 비교 표시한다

### OUT OF SCOPE — 아래는 절대 하지 않는다
1. 유료 API(Google Trends API, SEMrush, Ahrefs, Moz, SimilarWeb)를 호출하거나 API 키를 요구하지 않는다. WebSearch/WebFetch만 사용한다.
2. 점수 산출 없이 단순 검색 결과 나열로 응답을 종료하지 않는다. 반드시 6개 지표 점수와 Final Score를 포함해야 한다.
3. 수집 데이터 없이 추측으로 점수를 부여하지 않는다. 근거 데이터가 없는 지표는 N/A로 표기하고, N/A 지표는 가중 평균에서 제외한다.
4. 평가 주제를 15개 초과로 확장하지 않는다. 15개를 넘으면 사용자에게 범위 축소를 요청한다.
5. 투자 추천, 매수/매도 판단, 수익 예측, 포트폴리오 제안을 제공하지 않는다. 트렌드 데이터 분석만 수행한다.
6. 개인정보, 특정 인물의 사생활, 불법 활동 관련 주제를 평가하지 않는다.

### researcher 스킬과의 구분
- **trend-score**: 여러 주제를 동일 기준으로 **정량 비교·순위화**할 때 사용. 출력은 점수 테이블.
- **researcher**: 하나의 주제를 **깊이 조사**할 때 사용. 출력은 리서치 리포트.
- 판단 기준: 사용자가 "점수", "랭킹", "비교", "뭐가 더 뜨는지"를 원하면 trend-score. "자세히 알아봐", "조사해줘"를 원하면 researcher.

## 1. 입력 파악

사용자 요청을 아래 기준으로 분류한다:

| 입력 유형 | 예시 | 처리 |
|-----------|------|------|
| **주제 지정** | "AI agent vs MCP 트렌드 비교해줘" | 지정된 주제만 평가 |
| **자동 발굴** | "요즘 뭐가 뜨는지 알아봐" | Step 2로 이동해 주제 발굴 |
| **카테고리 지정 발굴** | "기술 분야에서 뜨는 거 찾아줘" | 해당 카테고리 내에서 발굴 |

**시장 범위:**
- 명시 없음 → 한국 시장 기준 (네이버, 한국어 소스 우선)
- "글로벌", "미국", "유럽", "일본", 영문 키워드, 또는 특정 국가/지역 명시 → 해당 시장 기준

**카테고리:**
- 명시 없음 → 범용 (기술, 비즈니스, 문화, 엔터, 라이프스타일, 사회 이슈 전체 포함)
- 명시 있음 → 해당 카테고리로 한정

## 2. 주제 발굴 (자동 발굴 모드)

사용자가 주제를 직접 지정하지 않은 경우, 아래 소스에서 후보 주제를 수집한다.

### 데이터 수집 쿼리 (WebSearch)

한국 시장 기준 예시:
- `"2026 트렌드 키워드"`, `"요즘 핫한 키워드 {현재월}"`
- `"네이버 실시간 검색어"`, `"구글 트렌드 한국"`
- `"Reddit trending Korea"`, `"Twitter trending South Korea"`
- 카테고리 지정 시: `"{카테고리} 트렌드 2026"`, `"{카테고리} 인기 키워드"`

글로벌 기준:
- `"trending topics {month} 2026"`, `"Google Trends rising"`
- `"Reddit popular today"`, `"HackerNews front page"`
- `"TikTok trending"`, `"YouTube trending"`

**최소 5개, 최대 15개** 후보 주제를 선별한다. 너무 포괄적인 단어(예: "AI", "경제")는 구체화한다(예: "AI 에이전트 프레임워크", "금리 인하 영향").

---

## 3. 주제별 데이터 수집

각 주제에 대해 아래 6개 지표의 근거 데이터를 WebSearch + WebFetch로 수집한다.

### Trending 지표 (단기 급상승 측정)

| 지표 | 약어 | 수집 방법 |
|------|------|-----------|
| 검색량 증가율 | SVGR | `"{주제} Google Trends"` 또는 `"{주제} 네이버 트렌드"` → 최근 1주/1개월 변화율 확인 |
| SNS 버즈 지수 | SBI | `"{주제} site:reddit.com"`, `"{주제} 트위터 반응"` → 최근 게시물 빈도·반응 수준 파악 |
| 뉴스 빈도 지수 | NFI | `"{주제} 뉴스 {현재월}"` → 최근 1주 내 주요 기사 수 및 헤드라인 빈도 |

### Steady 지표 (장기 안정성 측정)

| 지표 | 약어 | 수집 방법 |
|------|------|-----------|
| 안정성 점수 | STB | `"{주제} Google Trends 12개월"` 또는 `"{주제} 장기 트렌드"` → 검색 결과에서 "꾸준한 관심", "안정적", "횡보" 등의 표현과 그래프 설명을 확인. 급등/급락 언급이 적을수록 높은 점수 |
| 기반 볼륨 점수 | VOL | `"{주제} 검색량"`, `"{주제} 인기도"` → 검색 결과에서 절대 검색량 수준("수십만", "수백만")이나 주요 플랫폼(포털 메인, 앱스토어 상위) 노출 여부를 확인 |
| 계절성 규칙도 | SEA | `"{주제} 계절 트렌드"`, `"{주제} 연간 패턴"` → 검색 결과에서 매년 특정 시기 반복 언급(예: "매년 연말", "시즌마다")이 있는지 확인. 공식 이벤트/시즌과 연계되면 높은 점수 |

**데이터가 불충분한 경우**:
- 특정 지표의 WebSearch 결과가 0건이면 해당 지표를 N/A로 표기하고, 가중 평균 계산 시 해당 지표를 제외한 뒤 나머지 지표에 가중치를 비례 배분한다. (N/A 지표는 0점이 아니라 계산에서 제외되는 것이다.)
- 한 주제에서 N/A 지표가 3개 이상이면 해당 주제에 "데이터 부족 — 신뢰도 낮음" 경고를 표시하되, 가용한 지표로 점수는 산출한다.
- 한 주제에서 6개 지표 모두 N/A이면 해당 주제를 랭킹에서 제외하고, "데이터 수집 불가로 평가 제외" 사유를 리포트 하단에 기재한다.

## 4. 점수 산출

### 4.1 개별 지표 점수 (0~10)

각 지표를 0~10 정수로 환산한다. WebSearch 데이터에서 정확한 수치를 추출할 수 없는 경우, 아래 기준을 상대적 판단 가이드로 사용하되 **근거 한 줄**에 판단 이유를 반드시 기재한다.

#### SVGR (검색량 증가율)
| 점수 | 기준 |
|------|------|
| 9-10 | Google Trends에서 최근 1주가 직전 4주 평균 대비 3배 이상 급등, 또는 "폭발적 검색 증가" 표현이 복수 소스에서 확인 |
| 7-8 | 최근 1주가 직전 4주 평균 대비 2배 이상 증가, 또는 "검색 급증", "관심 급상승" 표현 확인 |
| 5-6 | 최근 1주가 직전 4주 평균 대비 1.2~2배 증가, 또는 뚜렷한 증감 언급 없이 꾸준한 관심 |
| 3-4 | 증가 추세 미미하거나 횡보, 낮은 수준 유지 |
| 1-2 | 검색 결과 5건 미만이고 최근 언급 거의 없음 |
| 0 | 관련 검색 결과 없음 (N/A 처리) |

#### SBI (SNS 버즈 지수)
| 점수 | 기준 |
|------|------|
| 9-10 | Reddit/X에서 1주 내 1000+ upvotes 게시물 존재, 또는 복수 플랫폼에서 동시 바이럴 |
| 7-8 | 1주 내 500+ upvotes 게시물 또는 활발한 토론 스레드 3개 이상 |
| 5-6 | 1주 내 관련 게시물 10개 이상이나 대부분 저반응(100 미만) |
| 3-4 | 관련 게시물 5개 미만, 반응 미미 |
| 1-2 | 최근 1개월 내 관련 게시물 1~2개만 발견 |
| 0 | SNS 관련 게시물 없음 (N/A 처리) |

#### NFI (뉴스 빈도 지수)
| 점수 | 기준 |
|------|------|
| 9-10 | 최근 1주 내 주요 언론사 헤드라인 10건 이상, 또는 포털 메인 뉴스에 노출 |
| 7-8 | 최근 1주 내 주요 기사 5~9건 |
| 5-6 | 최근 1주 내 관련 기사 3~4건 |
| 3-4 | 최근 1주 내 기사 1~2건 |
| 1-2 | 최근 1개월 내 기사 1~2건 |
| 0 | 관련 뉴스 없음 (N/A 처리) |

#### STB (안정성 점수)
| 점수 | 기준 |
|------|------|
| 9-10 | Google Trends 12개월 그래프가 거의 수평, 또는 "꾸준한 관심", "안정적 검색량" 표현 확인 |
| 7-8 | 12개월간 소폭 변동만 존재, 급등/급락 구간 없음 |
| 5-6 | 1~2회 눈에 띄는 변동이 있으나 전반적 추세는 유지 |
| 3-4 | 큰 변동 3회 이상, 또는 지속적 하락 추세 |
| 1-2 | 극심한 변동(특정 시점만 급등 후 소멸) |
| 0 | 12개월 데이터 없음 (N/A 처리) |

#### VOL (기반 볼륨 점수)
| 점수 | 기준 |
|------|------|
| 9-10 | Google Trends 비교 기준 대비 최상위권, 또는 월간 검색량 "수십만 이상"으로 추정되는 소스 확인 |
| 7-8 | 비교 기준 대비 상위권, 지속적 높은 관심 |
| 5-6 | 중간 수준의 검색 볼륨 |
| 3-4 | 니치 주제, 소수만 관심 |
| 1-2 | 매우 적은 검색량 |
| 0 | 검색량 데이터 없음 (N/A 처리) |

#### SEA (계절성 규칙도)
| 점수 | 기준 |
|------|------|
| 9-10 | 12개월 그래프에서 매년 동일 시기 반복 패턴이 명확히 관찰됨 (예: 블랙프라이데이, 수능) |
| 7-8 | 반복 패턴이 있으나 시기/강도가 다소 변동 |
| 5-6 | 약한 계절성 또는 패턴 판단 어려움 |
| 3-4 | 비계절적이나 불규칙 변동 존재 |
| 1-2 | 완전 비계절적, 불규칙 |
| 0 | 패턴 판단 불가 (N/A 처리) |

### 4.2 카테고리 점수

```
Trending Score (TS) = (SVGR × 0.40) + (SBI × 0.35) + (NFI × 0.25)
Steady Score (SS)   = (STB × 0.40) + (VOL × 0.35) + (SEA × 0.25)
```

가중치 근거:
- Trending에서 검색량 증가율(SVGR)이 가장 직접적인 수요 신호이므로 최고 가중치
- Steady에서 안정성(STB)이 지속성의 핵심이므로 최고 가중치
- 계절성(SEA)은 보조 지표이므로 가장 낮은 가중치

### 4.3 종합 점수

```
Final Score = (TS × α) + (SS × β)

기본값: α = 0.55, β = 0.45
```

사용자가 가중치 선호를 표현하면 아래 규칙으로 조정한다. 판단 기준: 사용자 메시지에 "인기", "핫한", "급상승", "떠오르는"이 포함되면 인기 위주. "꾸준", "스테디", "안정", "장기"가 포함되면 스테디 위주. 해당 키워드가 없으면 기본값 유지:
- 인기 위주: α = 0.70, β = 0.30
- 스테디 위주: α = 0.30, β = 0.70
- 균등 ("균등", "반반", "동일"): α = 0.50, β = 0.50
- **키워드 충돌** (인기 위주 + 스테디 위주 키워드가 동시에 포함된 경우): 기본값(α = 0.55, β = 0.45)을 적용하고, "인기/스테디 키워드가 모두 감지되어 기본 가중치를 적용했습니다. 조정을 원하시면 α, β 값을 직접 지정해주세요."라고 안내한다.
- 기본값 (선호 표현 없음): α = 0.55, β = 0.45

## 5. 출력

### 기본 출력: 대화 내 테이블

```markdown
# Trend Score Report
> 평가일: {날짜} | 시장: {한국/글로벌} | 카테고리: {범용/지정}
> 가중치: Trending α={값}, Steady β={값}

## 종합 랭킹

| 순위 | 주제 | Final | TS | SS | SVGR | SBI | NFI | STB | VOL | SEA |
|------|------|-------|-----|-----|------|-----|-----|-----|-----|-----|
| 1 | ... | 8.2 | 9.0 | 7.2 | 10 | 8 | 9 | 7 | 8 | 6 |
| 2 | ... | 7.5 | 6.8 | 8.3 | 7 | 6 | 7 | 9 | 8 | 8 |

## 주제별 상세 분석

### 1위: {주제명} (Final: X.X)
- **왜 높은가**: (1~2문장 핵심 근거)
- **Trending 근거**: SVGR {점수} — {데이터 출처 요약}
- **Steady 근거**: STB {점수} — {데이터 출처 요약}
- **주의점**: (있으면 기재, 없으면 생략)
```

### 파일 출력 (사용자 요청 시)

사용자가 파일 저장을 요청하면 아래 형식으로 저장:
- Markdown: `trend-report-{YYYY-MM-DD}.md`
- CSV: `trend-report-{YYYY-MM-DD}.csv` (랭킹 테이블 데이터만)

저장 경로는 현재 작업 디렉토리 기본, 사용자 지정 경로 우선.

## 6. /loop 연계

이 스킬은 `/loop`과 함께 사용하면 주기적 트렌드 모니터링이 된다.

사용 예: `/loop 6h /trend-score 기술 트렌드 한국 시장`

주기적 실행 시 이전 결과와 비교할 수 있도록, 파일 출력 모드를 자동 활성화하고 날짜별 파일명으로 저장한다. 이전 리포트가 같은 디렉토리에 있으면 랭킹 테이블에 변동 컬럼을 추가한다:

```markdown
| 순위 | 변동 | 주제 | Final | 이전 Final | TS | SS |
|------|------|------|-------|-----------|-----|-----|
| 1 | NEW | ... | 8.2 | - | 9.0 | 7.2 |
| 2 | +2 | ... | 7.5 | 6.8 | 6.8 | 8.3 |
| 3 | -1 | ... | 7.1 | 7.3 | 7.0 | 7.2 |
```

변동 표기: `+N` (N단계 상승), `-N` (N단계 하락), `=` (변동 없음), `NEW` (신규 진입), `OUT` (이전 존재했으나 이번에 탈락 — 리포트 하단에 별도 기재)

## 7. 에러 및 예외 처리

### WebSearch/WebFetch 도구 오류
- WebSearch 호출이 에러를 반환하면 (타임아웃, 네트워크 오류 등): 최대 2회 재시도한다. 2회 재시도 후에도 실패하면 해당 지표를 N/A로 처리하고, 리포트 상단에 "일부 검색이 도구 오류로 실패했습니다. 해당 지표는 N/A 처리되었습니다."를 명시한다.
- WebFetch가 특정 페이지에서 콘텐츠를 가져오지 못하면: 대체 소스를 WebSearch로 1회 추가 검색한다. 대체 소스도 없으면 해당 데이터 포인트를 생략한다.

### 사용자 지정 주제가 1개인 경우
- 주제가 1개이면 순위 비교는 불가하므로, 랭킹 테이블 대신 단일 주제 상세 분석 형식으로 출력한다. "종합 랭킹" 섹션의 테이블은 1행만 표시하고, "주제별 상세 분석" 섹션에 6개 지표 전체의 근거를 상세히 기술한다.

### 자동 발굴 실패
- 자동 발굴 모드에서 후보 주제를 1개도 수집하지 못한 경우: "현재 시점에서 해당 카테고리/시장의 트렌드 데이터를 충분히 수집할 수 없습니다. 검색어를 변경하거나 주제를 직접 지정해 주세요."라고 응답하고, 빈 리포트를 생성하지 않는다.

### 전체 주제 평가 불가
- 모든 주제에서 6개 지표가 전부 N/A인 경우: "모든 주제에 대해 데이터를 수집할 수 없어 리포트를 생성하지 못했습니다."라고 응답하고, 가능한 원인(검색어가 너무 신조어, 외국어 주제를 한국 시장에서 검색 등)을 1~2개 제시한다.

## 주의사항

- **데이터 한계 인정**: WebSearch 결과는 실시간 API 데이터가 아니다. "정확한 검색량 수치"가 아닌 "상대적 수준과 추세"를 평가하는 것임을 리포트에 명시한다.
- **출처 표기**: 각 지표의 점수 근거에 어떤 검색 결과를 참고했는지 간략히 밝힌다.
- **과신 금지**: 데이터가 부족한 지표가 3개 이상이면 "데이터 부족으로 신뢰도 낮음"을 명시한다.
- **편향 주의**: 한국 시장 분석 시 네이버 + 구글 양쪽을 확인한다. 한쪽만으로는 편향될 수 있다.
